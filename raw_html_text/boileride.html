Inspiration

Our members use the Facebook Purdue Ride Share group and we found it difficult to find rides by scrolling through all the posts. The group was tedious to navigate so we wanted to build an app to solve the problem of finding rides in a more user-friendly way.

What it does

Users can post rides and can lookup rides that are posted. This makes it easier to connect those who are in need of or can offer rides.

How we built it

Used Django and HTML/CSS/Python to create the webapp and store user data.

Challenges we ran into

None of us knew how to use Django, so we had to find tutorials to understand how to use it.

Accomplishments that we're proud of

We were able to create a prototype of the website and some basic functionality such as user login and the idea of how to post/view rides.

What we learned

How to use Django and team communication. How to stay awake while simultaneously code for 36+ hours.

What's next for BoileRide

We plan on continuing building it to create a viable product. We want to add more functionality such as search function and a map aspect as well as polishing some of the current functionality.
