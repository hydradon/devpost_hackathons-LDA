What it does

It is a game: users access the website and can take part in a game, adding one line of code at a time each. Users submit lines of code twitting to our twitter account (Engineering without orders) the line of code they want add or edit from the file. The result can be a program itself or simply a function (or methods) of the program. There is also another Github Repo in which you can see files people created.

How we built it

We built a front end with a text editor, and we accesses twitter api using python

Challenges we ran into

Struggling with the token of twitter API. Updating in real time the text file.

Accomplishments that we're proud of

Having the part regarding twitter api working

What we learned

We explored Twitter Api.

What's next for Engineers Without Orders

Syntax checking, malicious code checking. Make it more fun and interactive.
