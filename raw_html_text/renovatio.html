Inspiration

All of our team member really like to play multiplayer games. In our opinion the more players – the more fun it is to play. We were inspired by Capture The Flag, but we added our own twist.

What it does

It's a multiplayer game where two factions compete with each other. One side has to destroy the ancient statue, while the other has to renovate it and resurrect it, in the end.

How we built it

We used Unity3D to develop the client. Our server was written in Java and hosted on Heroku. The designer of our team used Photoshop CC, Illustrator, Blender.

We also used the following libraries: https://www.assetstore.unity3d.com/en/#!/content/27944 https://www.assetstore.unity3d.com/en/#!/content/45197 https://www.assetstore.unity3d.com/en/#!/content/48492 http://www.blendswap.com/blends/view/83588

Challenges we ran into

We decided to use http protocol for our multiplayer game, so we had to optimize the traffic (because http has a big overhead). Finally, we came to the solution, when requests are sent only when movement direction is changed. To implement this, we made an extrapolation component in Unity
