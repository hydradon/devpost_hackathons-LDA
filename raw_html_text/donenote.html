Inspiration

Too often our attention goes towards taking notes, instead of understanding what we're writing. That changes with DoneNote.

What it does

It automatically transcribes the words of your professor and summarizes them into a concise overview for future studying.

How we built it

We built it using Bing's API for Speech-to-Text Recognition, and we hosted it using Microsoft Azure.

Challenges we ran into

We originally tried making a java applet, but ran into issues implementing external APIs like Watson and Aylien.

Accomplishments that we're proud of

As an inexperienced team, we are proud of the way we adapted to using new languages and environments as we ran into issues.

What we learned

For many of us, it was our first Hackathon and our first time using technologies like javascript and Azure, so we learned a lot about how different technologies interact and the importance of adaptability.

What's next for DoneNote

Moving forward, we'd like to continue learning about these technologies and to implement the additional features that we had originally intended to be a part of DoneNote.
