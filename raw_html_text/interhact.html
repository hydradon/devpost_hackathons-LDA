Inspiration

Patients are bored and lonely in hospital.

What it does

Interactive experience for patients. Allows for patient-to-patient interaction through games and chat. Data collected from game playing can also be analysed by doctors to provide health metrics to monitor the patient’s progress. The problem we aim to solve with this technology is to eliminate the loneliness and boredom too often experienced by patients stuck in hospital whilst also giving doctors another channel of information related to their patient’s health.

How we built it

We used a prototype of a tic-tac-toe game and integrated it into the Spark framework. We used a node.js server to talk with the Spark system and Django server to store the relevant identifiers for users to be able to refer to them in Spark.

Challenges we ran into

Integrating the cisco spark api with our application

Accomplishments that we're proud of

Integrating different technologies into one system

What we learned
