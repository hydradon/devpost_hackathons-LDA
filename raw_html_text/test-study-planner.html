Inspiration

The difficulty on deciding how much time to spend on each section studying for a test.

What it does

Determines how much time to spend on each section depending on how many wrong problems were done on homework, the time spent studying each day, and the number of days before the test.

How I built it

I used android studio and coded the app in java. I used parse to code the login part of the app. I had the user input answers to the questions on the screen. Then i multiplied hours of say studying with number of days to get total days. Then I used proportions and ratios to calculate the time for each section. I calculated it into minutes to make it easier for the user to read and get information.

Challenges I ran into

There was a problem while using array adapters to fill the listviews of android. Another problem was developing its backend. I couldn't get how backend worked so i stopped trying to do it.

Accomplishments that I'm proud of

Proud that I finished the basic app that can work on a local device and save all the test plans on the local device.

What I learned

I learned that trying to develop backend to an app is harder then it seems and that androids require tons of permissions compared to coding java on a laptop.

What's next for Test Study Planner

Develop its backend so that the information can be accessed from multiple devices. Have the ability to check of the test study plans that have been finished.
