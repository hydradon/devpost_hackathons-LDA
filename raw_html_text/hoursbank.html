Inspiration

HourBank was originally pitched to us during a presentation of many community projects

What it does

It is a mode of sending requests for labor and trading time spent on labors

How we built it

We built it as a single-paged javascript app that manages the data inside localStorage; the data model was designed in such a way to be compatible with SQL databases -- localStorage was used here to make it easier to deploy at a hackathon.

Challenges we ran into

We struggled a lot with creating the database to store information about people

Accomplishments that we're proud of

Making everything working in such a short time span

What we learned

We learned a lot about working with servers

What's next for HoursBank

Making it more secure and adding depreciation of currency
