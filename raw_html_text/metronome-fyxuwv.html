Inspiration

When the Twilio representative gave his tech talk, we were immediately interested in using the Twilio API. It was hard to understand the documentation at first, but as a group, we were able to comprehend the complexities of this API. We figured out a solution for music playlists for social events but using this API to vote for the audience's favorite song.

What it does

Our product receives SMS text messages and displays results on a website table. Our product has potential as a music queue for DJs and radio stations.

How we built it

We used HTML, CSS, javascript, python, json, flask, and Twilio API.

Challenges we ran into

Trying to use flask and display the flask results on the web page.

Accomplishments that we're proud of

Successfully finished a majority of the project in about 24 hours.

What we learned

We should try to plan ahead by preparing all dependencies for a project beforehand.

What's next for Metronome

We are planning to spread our creation through social media. We want more users from the demographics of music curation to use our product. We also hope to use other API to help improve our current website.
