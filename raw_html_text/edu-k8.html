Inspiration

The lack of consistent wi-fi limits the resources available for students to learn.

What it does

We want to fix this issue by giving students a variety of videos that can be locally stored and allowing them to sync up with they get wi-fi access.

How we built it

It's a local html-based environment that has some videos pre-stored. We are using a Google api to sync in new videos when wi-fi access is given.

Challenges we ran into

Connecting Google api through python.

Accomplishments that we're proud of

A multi-page web-page that is mostly functional.

What we learned

How to using Google api and web-design.

What's next for EDU-K8

Having a larger video library and moving to a real piece of hardware that can connect to server more freely than using wi-fi.
