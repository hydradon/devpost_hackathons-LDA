Inspiration

Nuestra inspiración para crear la aplicación fueron las constantes alarmas que se generaron a partir de las noticias reportadas del volcán popocatépetl. Pensamos que es necesario tener mejores concideraciones a la hora de que una población sufra una catastrofe natural.

What it does

Al accionarse un estado de emergencia (cómo desastres naturles) se envía la localiazación del usuario entre familiares para que conzcan su paradero. Esta acción puede ser ejecutada por el usuario (reportandose así mismo) ó por una institución gubernamental la cual identifique una zona de riesgo y emita la ubicación de todos los usuarios pertenecientes a esa zona. Muestra también los albergues y centros de acopio ceracanos. Proporciona información sencilla para que el usuario sepa a dónde refugiarse.

Sí las redes inalambricas llegarán a caerse la aplicación esta preparada para guardar la ubicación, para después que tenga internet se puedan enviar todos los puntos por los que pasó.

How we built it

Nuestro equipo de desarrollo está conformado por cuatro personas, dos de ellas dedicadas a back-end (Isidro y Miguel). Relacionadas con MongoDB y loopback levantando una api rest que se pudiera conectar con el desarrollo en front-end; Esta parte realizada por Juda y Daniel codificando de manera nativa para android.

Challenges we ran into

El desarrollo de nuestra aplicación tuvo una complicación a causa del internet, en su momento contratamos de lineas teléfonicas con datos para poder continuar con nuestro trabajo. Otra de ellas y la manera de trabajar con Android y la inclusión del consumo de servicios de una API tipo REST.

Accomplishments that we're proud of

Estamos satisfechos de haber trabajado como un verdadero equipo, apoyándonos en todo momento y de haber cumplido con el objetivo que nos planteamos inicialmente de los alcances que tendría la aplicación al final del evento.

What we learned

Aprendimos a integrar una aplicación móvil con servicios REST principalmente y a trabajar de una forma organizada y repartiendo tareas puntualmente entre los miembros del equipo. Además parte de nuestro aprendisaje fue manejar la tensión, la falta de sueño y la presión de desarrollar una aplicacación la cual se pueda considerar un buen prototipo para mostrar.

What's next for 32:Tezca

El proyecto pretende integrar la funcionalidad expuesta con una interfaz web desde la cual las autoridades puedan lanzar las alertas de riesgo.

Dar funcionalidades off-line como información sobre los lineamientos a seguir durante las principales eventualidades de riesgo como terremotos, huracanes, erupciones, etc.

En el futuro nos gustaría integrar también un módulo basado en reconocimiento de razgos antropométricos (como el facial o dactilar), que nos permita enviar notificaciones a familiares en caso de que ingresen a un hospital o tengan un percance que una autoridad pudiera reportar a los familiares del afectado y que estén registrados en nuesto sistema.
