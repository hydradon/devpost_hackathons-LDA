Inspiration

My inspiration was to build an app that enables safe and efficient trips for Detroit citizens.

What it does

The main features are:

  • Finding route (safest, fastest or shortest) for specific transit type (drive, bicycle and walk)
  • Show routing summary
  • Sending route to Google Maps for navigation
  • Adding stop from current location, point on map, address, favourite location
  • Adding point barrier
  • Bypassing current traffic incidents
  • Sharing user's location with other users
  • Showing additional layers (average incident density, current traffic conditions and friends' locations)
  • Creating summary of incidents in neighbourhood from selected stop

How I built it

I used Open Street Map data for offline routing as well as incident data from Detroit City as for setting routing cost attribute. I used also ArcGIS Online for sharing the outputs

Challenges I ran into

Creating the appropriate network dataset for routing was not an easy beast.

Accomplishments that I'm proud of

What I am the most proud of, is that the routing analytics are working completely offline. However, they can be enhanced by current traffic conditions and other incidents in Detroit.

What I learned

Well, I learnt how to use ArcGIS Runtime for Android 100.0.0 for routing purposes. I got familiar with Android Studio as well.
