Inspiration

We had a long flight from London to Los Angeles and we noticed that the corridors on the plane were always crowded with people waiting for their turn to use lavatory. This is was a problem for the flight staff, as well as inconvenient for the passenger.

What it does

EasyP is a virtual lavatory queue and thus it keeps the corridors empty. Every passenger can reserve the lavatory using the EasyP app from his app or the onflight entertainment terminal.

How we built it

We used the panasonic aviation API to gather relevant passenger information, such as the seat number and name. And used the device messaging API by thingSpace to connect a sensor that reads out the state of the lavatory door.

Challenges we ran into

Designing multiple devices with interaction was a key challenge that needed to be solved. And gracefully handled by the publish and subscribe protocol of the device messaging api. Connecting and controlling the inflight media was handled by the panasonic avionics api.

Accomplishments that we're proud of

Multi device integration combined with lean web services development suitable for all kinds old and new mobile devices.

What we learned

Do not nest to many closures...
