Inspiration

Recent media has portrayed surprise bills and out-of-network charges. People can help shape federal regulations to prevent these situations from being the responsibility of the consumer.

What it does

We provide a platform to make proposed federal rules for healthcare more accessible. We score comments with a sentiment to show the polarity. This helps guide users to understand whether comments are yay or nay.

How we built it

We built the application using Python Flask, React.js and Heroku.

Challenges we ran into

Our major challenge was not being able to get the regulations.gov API immediately, since the government only works on the weekdays. We side-stepped this integration issue by download the data.

Accomplishments that we're proud of

We're proud of creating our own API using the downloaded data.

What we learned

We learned that there is a lot of opportunity to do natural language processing and that this is only the beginning of the open project.

What's next for HealthCivic

We want to add more features as we get more information about how users are interacting with the platform. For example, we could build a bot that answers questions about the rule and comments, and automate submitting comments.
