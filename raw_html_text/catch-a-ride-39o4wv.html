Inspiration

When considering all of the options for local travel (i.e. Taxi, Uber, Lyft, Friends), we considered the properties of managing all of this information in a single location. This type of application does not necessarily exist, especially within the mobile environment.

What it does

It gather information from the user (location, and destination preference), along with the details for various ride options from Uber and Lyft, and recommends the current rides available at the given time

How we built it

We built it using Android Studio and using the Uber, Lyft, Google Maps APIs.

Challenges we ran into

Learning how to make the Android app, along with Wi-Fi issues.

Accomplishments that we're proud of

The overall user interface and functionality of the application is remarkable, especially within the 24 hour time period.

What we learned

We learned a lot about Android development and utilizing various APIs on a mobile platform.

What's next for Catch-a-Ride

Further development within the user interface (button development, map interactions, and how further information can be useful for the user).
