Inspiration

We found that the original idea of making a "pokemon" game would take too long to implement. We found ourselves at the point of being able to do object detection, and decided to make a maze out of it.

What it does

It automatically generates a random maze and requires to user to solve it. However, solving it is not as easy, since the orientation of the maze constantly changes

How we built it

We used the Recursive Division algorithm to generate the maze and matrices to keep tracking of where the play can go, as well as the start and end

Challenges we ran into

We had problem sorting the data of scores

Accomplishments that we're proud of

Implementing a maze algorithm and make others "think" when they are solving the maze

What we learned

We learned the Recursive Division algorithm We learned about object collision through Matrices

What's next for MazeMaster

Attempt to make it into an app
