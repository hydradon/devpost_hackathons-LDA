Inspiration

When in high school it was a challenge to book tours with universities. Each university had their own website and you had to keep track of their dates manually. Moreover, it was hard to coordinate and plan multiple visits all at once.

What it does

Brings all the university tours in the same place, provides the user with filterable search, allowing them to book multiple university tours all at once. Furthermore, it provides suggestions for accommodation, and transportation means.

How we built it

With lots of coffee! Not a lot of sleep!

But seriously, we first created story boards on a whiteboard. Jessica then learned to use sketch within a couple of hours and got started on the mockups. Arshan got started on learning React, and putting together a template project. Sasha put together a simple flask API server that acts as a middleware between the front-end application and Accenture's API's. This design was chosen to have the flexibility to expand on Accenture's API using Firebase, or Azure.

Challenges we ran into

Authentication is actually pretty hard if you try learning it in two hours! Same thing goes for React! Story boards and UX decisions are very hard to come up with, as there are a lot of different paths in the application, and time needs to be spent to ensure a simplified model of the userflow.

Accomplishments that we're proud of

We learned a lot!

What we learned

Arshan learnt react, and how authentication works. Sasha learnt backend web development, flask, and setting up a storage using firebase. Jessica learnt how to use sketch, and was able to come up with storyboards and mockups.

What's next for Accentours

Being able to create events, and splitting the application into "Admins" and "Users". ReactNative can be used to port the application to mobile.
