Inspiration

One of our team members' parents discovered restaurants in San Francisco by trying out a new cuisine each week, working their way through the alphabet. We wanted to share this way of exploring local restaurants with other San Francisco residents and visitors.

What it does

EatLocalAZ shows the user cuisines for any letter of the alphabet. Once they select a cuisine, the user can peruse a list of local SF restaurants serving this type of food.

How I built it

Node/Express back-end, Angular for front-end display, and Ionic to get it running on a smartphone.

Challenges I ran into

Creating users/login: Auth/Oauth was too time-consuming. Getting the app to run on our phones was difficult.

Accomplishments that I'm proud of

Learning enough Ionic/Angular to get by Building the node server/yelp API wrapper in about ten minutes Deploying server to Heroku

What I learned

Start with the absolutely smallest possible MVP, and only add to that when you're sure you have extra time.

What's next for EatLocalAZ

We're going out to eat, locally, after the hackathon! Oh, and we may work on the app more, too. Depends on how good our local dining experience is ;)
