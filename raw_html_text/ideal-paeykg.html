Inspiration

More sustainable living - realized how much food we throw away and when researching the topic, were surprised by how many co2 emissions (in kilos) were created during the production of food

What it does

A web-app that takes information from scanning barcodes or manual entries of products, adds it to a database and shows the user their current 'foods' with their respective co2 emissions, date till expiry and recipes.

How we built it

Python-flask to set up the web-app. HTML and CSS styling for the web-pages. The Database was created using mongodb

Challenges we ran into

Tried to enable sign up through Github and twitter , got the links to work and redirect to their sites but had issue with the tokens and creating a user once their login was authorized. Unfortunately had to remove this feature for production due to time restrictions. Multiple issues with CSS styling. Google Cloud Platform. Pycharm.

Accomplishments that we're proud of

The main, login and sign up pages. Made the web-app live using Google Cloud.

What we learned

Learnt how to manipulate and create HTML and CSS How to use mongodb to create the server How to store the user login details and hash the passwords How to interact with Google Cloud Platform and make the web-app live

What's next for ideal

Complete the rest of the web-pages and integrate more APIs to trawl through more databases for food items. Add in extra features to the site. Could also include Car Miles Equivalent on top of the Co2 emissions. Incorporate adding more barcodes, not all work at the moment/ are in the current APIs, to reduce manual entry. Web notifications. Create a logo and integrate common overarching theme for the site.
