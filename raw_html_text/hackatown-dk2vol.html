///////Pas le temps de niaiser\\\\

La vie est trop courte pour passer son temps dans les files d’attentes.

///////Inspiration\\\\

Las de patienter à chaque fois que nous voulions faire une activité, nous avons voulu créer une solution permettant à tout le monde de profiter du temps perdu pour découvrir les alentours.

///////Utilisation\\\\

Cette application permet à n’importe qui de créer une file d’attente virtuelle que ce soit pour un commerce, un musée ou même un particulier. Un QR code est alors généré et permet aux personnes qui désirent entrer dans la file d’attente de le scanner pour s’ajouter à la file virtuelle.

L’application sur le mobile de cette personne lui indique alors le temps d’attente restant estimé. Elle lui propose également des points d’intérêts alentours suivant le temps d’attente restant. Ces points peuvent être triés par type (monuments, restaurants, etc…) permettant ainsi un large choix d’activités. Lorsque le temps est bientôt écoulé, une notification est envoyée sur le mobile du client, lui indiquant de se présenter à son tour.

Du côté du commerçant, la file est affichée ainsi que la position de chacun des clients. Il peut régler le temps moyen entre chaque client afin de créer l’estimation envoyée à chaque utilisateur. Lorsque le client arrive, un QR code est généré sur son mobile et doit être scanné par le commerçant afin de confirmer son identité et de rafraîchir la file virtuelle.

///////Technologies\\\\

Nous nous sommes basés sur des “progressives web apps” dans le but de garantir la meilleure expérience possible aux utilisateurs. Ainsi, une personne utilisant ponctuellement ce service se contentera de l’utiliser à travers un navigateur tandis qu’un utilisateur régulier pourra facilement l’ajouter à son écran d’accueil mobile ou navigateur afin de la lancer rapidement, en toutes circonstances.

Cette technologie permet de plus le déploiement d’application rapides, fonctionnelles et esthétiques quel que soit le support (Android, iOS, web) afin de toucher le plus d’utilisateur possible et de faciliter son utilisation. Les parties serveur et base de données sont basées sur le service Firebase de Google qui permet une mise à l’échelle facile et transparente, quel que soit le nombre d’utilisateurs.

///////Défis techniques\\\\

Les défis que nous avons rencontrés sont principalement liés aux technologies utilisées. En effet, bien que nous soyons convaincus que dans notre cas la meilleure expérience utilisateur passe par l'utilisation des “progressives web apps”, nous avons dû apprendre à mettre en place tous les éléments nécessaires à son fonctionnement et celui de Firebase en ayant peu d’expérience avec ces technologies.

///////Améliorations futures et plan commercial\\\\

L’application que nous avons réalisé contient les fonctionnalités de bases que nous avons voulu mettre en place. Il est néanmoins nécessaire de raffiner le code afin de garantir la meilleure expérience utilisateur possible.

Nous avons également prévu d’ajouter des options premium pour les gros commerces qui permettront de promouvoir les activités touristiques ou restaurants alentours ainsi que d’avoir une meilleure gestion de la file des utilisateurs.
