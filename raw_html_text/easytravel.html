Inspiration

Creemos que el reto de Reservamos era muy interesante por que es una de las bases de datos mas grandes de Mexico y queriamos darle el uso mas innovador que pudieramos imaginar en unas horas.

What it does

Elige un paquete de viaje por ti, completamente basado en preferencias y preguntas muy basicas. Hace el trabajo pesado para que el usuario pueda disfrutar de la comodidad.

How I built it

Primero aprendimos a usar GraphicsiQL para entender la API. Vimos que datos nos faltaban para armar nuestra base de datos. Empezamos los diseños en illustrator, procediendo a realizarlos en HTML y CSS sin ser funcionales aun. Mientras se hacia eso, tambien se trabajaba en construir el backend para que al final solo se conectara todo. Tambien, se dieron retoques al diseño al ultimo momento.

Challenges I ran into

Necesitabamos informacion que la API no nos daba, como hoteles, museos, etc. Decidimos entonces, empezar nuestra propia Api con una base de datos creada por nosotros. Por el tiempo solo llenamos pocos datos pero es posible hacerla mas grande y combinada con la API de reservamos creemos que tienen un gran potencial.

Accomplishments that I'm proud of

Logramos a tiempo perfecto terminar el proyecto sin dejar atras las actividades que se realizaron en el Hackathon. Aprendimos sobre APIs y reforzamos cosas que ya sabiamos.

What I learned

El principal aprendizaje fue utilizar la API de Reservamos con GraphicsQL. Es una plataforma en lo ultimo de desarrollo de APIS, sabiendo que muchas grandes empresas en linea como Facebook o Twitter estan migrando muchos servicios a este sistema.

What's next for EasyTravel

Creemos que es posible empezar nuestra propia pagina a partir de este concepto, pues es un diferenciador muy importante a cualquier otra de paquetes vacacionales que ya exista.
