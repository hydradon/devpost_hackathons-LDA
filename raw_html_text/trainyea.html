CTA Capacity Tracker System

Inspiration

The hackathon theme set the basic idea and we bounced ideas to each other mostly revolving around issues that we had with the CTA.

What it does

This Capacity Tracker System keeps track of train congestion down to the individual carts

How we built it

Back-end was built on Java, server on MySQL, and front end uses php, JavaScript, and html

Challenges we ran into

Structuring how our overall system would flow with each other back-end to server pushing php to JavaScript

Accomplishments that we are proud of

We learned how to program in languages that we were unfamiliar with. We came together and managed to manage each other as a team

What we learned

Learning different languages How to push to a SQL database from java

What's next for the CTACTS

If this idea were to be developed further we would actually install sensors on the CTA lines and maybe even counters that more accurately keep track of the people
