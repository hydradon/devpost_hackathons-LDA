Inspiration

A typical digital company has a backend Server team that creates APIs and a frontend UI Client team (Android, iOS, Web) which consumes these API to present to customers. This requires the APIs to be defined before the clients can start development work, and if not, leads to a lot of back and forth conversations (and frustration) between teams.

I faced this problem first hand and frustrated with all the back and forth, decided to build Apimint. With Apimint, you can easily define API with mock data which are immediately available over internet.

What it does

Apimint simplifies the collaboration process by providing,

  • A mock API server - The clients only need a façade to consume data. A mock API unblocks Client teams and allows Server teams to replace mock endpoints with actual endpoints seamlessly.
  • SDK Code Generation
  • Try APIs from within JIRA
