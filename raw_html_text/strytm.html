Inspiration

The thought originally came from fruit ninja, a mobile game where you cut fruit and dodge bombs to earn points. But instead of cutting fruit, what if you could cut words that build into a story? & with that, Fables were born.

What it does

It allows a user to create Fables. Words are cycled on the screen and the user selects their path by opening their mouth. Similar to ad-libs. Based on the fable they create, different 3D objects appear in the view, allowing you to create & live in your own fable.

How we built it

We used Spark AR Studio to create the AR experience and Javascript to change properties on those objects.

Challenges we ran into

Learning the Spark AR API to interact with objects and setting up monitor subscriptions.

Accomplishments that we're proud of

We are proud that we were able to get the MVP functionality. Our 3rd team member also didn't show up, which forced us to build the product (& chemistry) fast.

What we learned

Learned how to use Spark AR with no prior experience. We create objects and interact with these objects using Javascript.

What's next for Fables

Implement functionality to share the fable with your friends. Also, it would be cool to change it from opening your mouth to select a word to a gesture, similar to fruit ninja. Instead of cutting fruit, you'd cut the word that you want to go further in the fable. This gamifies the experience even further, allowing users to work harder (& smarter) for the fable they want.
