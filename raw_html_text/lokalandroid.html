Inspiration

We wanted optimize the inefficient social groups students use in a large volume on campus

What it does

LOKAL is an event planer app that uses the google maps API and Fire base to update a map in real time with events posted by users (Sports, Studying, Bars, Parties, Charity events.

How we built it

We worked backwards from the user interface by developing all our separate events in Android studios, The Functionality uses simple user registration to create the user a an account, stored on Firebase. From their lat and lon given by googles API a radius of 6km is mathematically generated around them returning all events within the circumference of the circle. users also have the ability to filter events to there liking or create their own.

Challenges we ran into

The majority of our group was new to Firebase which originally slowed our progress until we learned the ropes.

Accomplishments that we're proud of

We very proud with the possibility of our apps practical use, it has a lot of potential. We came up with to many features for just 36 hours, we hope to on our own time add to our app features such as Uber pricing and message boards as soon as we catch up on sleep.

What we learned

We all gained new knowledge of XML and deeper understanding/optimization of Firebase as well as using real-time processes for the first time. We enhanced our understanding of the Java language, connecting APIs, and working in with the Android SDK .

What's next for LokalAndroid

LOKAL is going to make its way to IOS as soon as possible accompanied with many planned features, Uber API, our own server, professional graphics .. etc
