Inspiration

We wanted to leverage as many Montreal datasets as possible, while still trying to make sense of them!

What it does

Search an address - we'll pull up the info about the good and the bad. Walk score, bike score, crime stats, nearby resto health violations.

How we built it

We gathered and stored a bunch of publicly available data in graphQL. Then, we used Node-RED to quickly set up a comprehensive API, and funneled that info into a pretty simple react native app with google maps!

Challenges we ran into

It was the first time using GraphQL for us, so getting used to the querying language was kinda new!

Accomplishments that we're proud of

The app looks pretty good.

What we learned

GraphQL and Flow-based programming with Node RED

What's next for Loco

Right now, the app almost seems too pessimistic! We'd like to add info about closest metro stations and possibly time of travel to a user's point of interests (work and school)
