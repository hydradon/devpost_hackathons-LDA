Inspiration

Motivated by Wakefern's community challenge, we wanted to build an app/website for ShopRite that increases consumers' awareness about the products they're buying.

What it does

Our app/website provides additional features that allow users to compare similar products in terms of cost, dietary restriction, and health. We also created graphs that provide visual aid on where products come from (US imports) to satisfy the growing trend of sustainability and awareness.

How we built it

For our app, due to our limited coding knowledge, we focused on user experience and created a prototype through Adobe XD. Our website was built using HTML, CSS, Javascript, Bootstrap, and d3.js while our graphs were created using Tableau. The dataset for our visualizations is from the United States Department of Agriculture .

Challenges we ran into

Learning how to use Adobe XD, HTML, CSS, Javascript, Bootstrap, and d3.js. Finding relevant data that consumers would be interested in, including food sources, supply chain, and locations of local farms. Finding a project that all of the members can contribute to as we have varying backgrounds and coding skills. We also discovered that this proposed update would require a way to standardize the various product sizes when comparing similar products. We were not sure what the best way to this that would reflect the most important concerns of shoppers and consumers (total price vs. price per ounce).

Accomplishments that we're proud of

Finishing the project on time, learning how to use various software, addressing relevant user concerns

What we learned

We learned how to work as a team, and how to use Adobe XD, HTML, CSS, Javascript, Bootstrap, and d3.js.

What's next for ShopRight

Our main goal would be to make a working prototype of both the app and website. This would require additional back-end coding knowledge as well as permission to use ShopRite's API and other existing frameworks. Once complete, we can start conducting research/beta testing to determine the user's true pain points with the current system and to make changes accordingly.
