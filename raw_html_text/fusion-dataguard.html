Inspiration

While working back in the day at Invest and currently in Markets, I understood there are many heterogeneous solutions to cover data integrity, alerting, barrier monitoring, etc. We had a dedicated project with an Invest client to provide checks around their ecosystem to verify interfaces, market data integrity, statics setup, compliance, barrier and position monitoring, etc. That was successful and quite fruitful, now the future is to provide the same checks using OpenAPIs

What it does

It is a monitoring tool for Front-Officers, Middle-Officers, Back-Officers. Provides real-time alerting on market data integrity, statics wrongly setup, compliance, customers KYC, position and barrier monitoring, etc. This way you control the data your core is fed with. It allows a flexible rule engine to dictate what alerts you want to receive and on which objects they apply

How we built it

We built it using OpenFin as the UI and microservices in the back-end, for calling Finastra OpenAPIs, calling the rule engine service and computing alerts to be sent to the front-end.

Challenges we ran into

Using different APIs on different channels (B2B, B2E) meant different types of authentication mechanisms. There were also problems with understanding the APIs and what data they provide, as there is some incomplete setup on the sandboxes. Needing more APIs is a clear must for the DataGuard to expand.

Accomplishments that we're proud of

Using OpenFin technology for the UI, making the app run smoothly with a kicking pitch. Also integrating iwatch notifications was cool as it gave a sense for IoT

What we learned

How to use the platform, APIs and create an app. Also the entire business need around the DataGuard idea

What's next for Fusion DataGuard

We have a clear roadmap that, if invested into, can smoothly productize the app. More APIs will be required to offer a complete solution. See our last slide for what's next! Thanks!
