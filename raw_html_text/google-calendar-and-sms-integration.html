This project was originally much more complicated. It involved a reservation system and an integrated website, where people could pick and reserve open rooms ahead of time or in the current moment. Eventually we pivoted ourselves and took a different more simplistic approach that we felt had great potential. This current approach focuses around the use of Google Calendar with the SMS from a phone number on Twilio. This number will text employees 5 minutes before their scheduled meeting. To build it, we took multiple steps using Twilio to send the text messages, and read through many documents in order to write the code involving Google API. We started running into complications involving the interactive website, which seemed too difficult for two people to manage in 24 hours. We also ran into problems with ngrok and the accessing token for the Google API. We are proud to have created a functioning program that seemed so simple but had proved very difficult. Nuo gained a refreshing experience of web development and Chris learned how to use applications such as Twilio and new text editors in the process. We are unsure about the future but may find ourselves competing in many more competitions.
