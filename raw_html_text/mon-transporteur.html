Inspiration

I take daily public transportation and there are multiple people waiting to take bus, metro, train,... It would be helpful to enable a multimodal voice first UXcustom skill to request the next schedules, in order to reduce these time consuming queues

What it does

It provides travellers next schedules for French Paris Public Transport (Metro, Bus, Suburb Train,...) per line/station

How I built it

It is a Custom Skill built with Node JS, AWS Lambda, AWS S3 (images storage) and interacts with Open Data API

Challenges I ran into

Integrate Alexa Presentation Language document/data sources to enable multimodal for multiple devices with display/touch screen like Echo SPOT, and Medium/Large Hubs but also Extra Large TV. In particular, I have used APL features to display/browse/touchSelect the dynamic list of data (transport modes, lines, stations)

Accomplishments that I'm proud of

Multimodal Custom Skill : Voice First and with mutimodal (display, touch screen,...) features with Alexa Presentation Language, UX equivalent to Display (Template) approach

What I learned

How to implement Custom Skill with Alexa Presentation Language recent technology
