Inspiration

Nous nous sommes inspirés de plusieurs données non utilisées par la ville de Montréal, mais pourtant facilement accessibles au public afin de pouvoir trouver cette idée extraordinaire.

What it does

En ouvrant l'application mobile, celle-ci te localise immédiatement et t'affiche une variété de panneaux, incluant ceux de stationnements interdits dans ton entourage. L'application permet aussi de faire une transition, selon l'utilisation d'un véhicule ou d'un vélo, en affichant les emplacements proches d'arceaux de bicyclette. Avec cette solution ingénieuse, ne crains plus jamais les amendes de stationnement, ni le temps perdu à chercher un endroit où barrer ton vélo.

Bred, cette application te rends plus à l'aise avec ton choix de stationnement!

How I built it

Nous l'avons créé avec React Native, un framework de Javascript qui permet de créer des applications mobiles multiplateformes. Nous avons aussi utilisé Expo pour visualiser les modifications apportées au code en temps réel C'était notre première utilisation de cet environment de travail.

Challenges I ran into

  • Réussir à cibler l'idée du projet
  • Gérer et traiter de très grands fichiers (plus que 100mb)
  • Se familiariser avec React Native
  • Expo fonctionnait pas toujours bien (crash souvent)
  • Rester réveiller

Accomplishments that I'm proud of

Nous sommes fiers d'avoir un prototype fonctionnel sur lequel il est possible de démontrer l'efficacité de notre produit.

What I learned

Nous avons appris beaucoup d'information sur le langage Javascript et React Native

What's next for ParkingBuddy

  • Augmenter la performance en ajoutants les données à une base de données GraphQL qui permettra de cibler des informations spécifiques
  • Intégrer le produit dans le tableau de bord des nouvelles voitures
  • Adapter le produit de sorte à ce qu'il puisse être fonctionnel en réalité augmentée
