Inspiration

Alessio talks, on the phone, with his dad when he is on the road (which is a lot). Many times they stay speaking to noone because his last lost the signal.

What it does

It tracks the user and when about to lose the signal shows a warning.

How we built it

We used HTML and JS to build the app using cordova and python for the server that serves the signal intensity information.

Challenges we ran into

We had never built an Android app using cordova before. We had minor difficulties with API call and google maps API. Finally we had issues with JavaScript and the webpage that is the app, as we are not proficient with these technologies.

Accomplishments that we're proud of

It works! It's been a lot of work but we got the MVP working nice and smooth.

What we learned

How to get an API working with Python and how to create a mobile app from a web-based one using phonegap and cordova.

What's next for Cellphone signal warning

It'd be cool to not only download information from the server but also update it with information from users. This way we'd ensure up-to-date information. But the next main step is to make it a background app, so that the user can be on a call or using the phone as a GPS navigator and get an overlay warning about the signal.
