What's next for Customer-Transaction-Prediction

We are inspired by the Santandra kaggle challenge and Citi Bank challenge. The system will predict a specific customer's next transaction. If it is going to be a deposit or withdrawal. We used Machine Learning technique Gradient Boost/GBM, basic html for the UI, and Python. We ran into sytem requirements problems and run time problems. Our system is 89% accurate. We learned to extract information from a set of data. In the future we would like to advance the UI, and we will expand the data range.
