TL DR.

Incentivized Torrent Client A minimum viable product with the following specifics: A simple torrent client with an associated ethereum account (best if serves directly from a browser, refer: webtorrent) 2 peers - one serving a file (seeder) and the other downloading a file (leecher) For each byte transfer to happen, the leecher transfers a small amount to seeder’s account and the seeder in return allows the transfer of the byte from their storage A faucet providing test tokens - that enable the transfer - can be used for this prototype.
