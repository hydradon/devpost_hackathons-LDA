Inspiration

Lack of innovation by 3rd party developers in the fin-tech space

What it does

Allows 3rd party developers to 'handle' sensitive financial data to build their own apps

How we built it

We built an encryption system on the blockchain which serves as the primary backend for our platform.

Challenges we ran into

Getting homomorphic encryption to work on the blockchain

Accomplishments that we're proud of

Building an actual MVC, an end to end demonstratable-enough product to pitch and convey our idea

What we learned

Quite a lot of Solidity(Smart Contracts), and you learn some js everytime you write some ;)

What's next for Pay Store

Getting out with a js library that allows rendering of 3rd party apps, so they have flexibility with design. As per the concept, possibilities of this architecture extend to any sensitive data (insurance, health, etc).
