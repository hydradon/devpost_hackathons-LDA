Our inspiration

The current payments process is manual and time consuming. We wanted to build something that enables businesses to pay their invoices more efficiently.

What it does

&Pay is a B2B platform that streamlines the invoice payments process. B2B users will upload invoices which will automatically be sent for payment authorisation. The authoriser can then authorise payments.

How we built it

We used Microsoft Power Apps to build the front end app UI and added 'Flows' to enable to functionality of the wireframes. We used Microsoft Vision API to read invoices from an image/ pdf and create the payment request. We used the FusionFabric.Cloud corporate banking APIs to integrate our app with the existing Finastra sandboxes.

Challenges we ran into

Linking the front end power apps UI with the flows for the demo. Not having a free Microsoft Vision API to convert an image to JSON. Understanding our users as we do not have experience in corporate banking. We needed to quickly understand the issues faced by corporate users who send payments regularly. We had to pivot our idea to meet the needs of a different use case. Reduced manpower!

Accomplishments that we're proud of

Being a team of two with limited technical abilities but we had help from superman, Greg at Microsoft!

What we learned

How to build a mobile app and link the front and back ends using Microsoft Power Apps. How to use Microsoft Vision API.

What's next for &Pay

We would like to add functionality that allows tagging of transactions. This will deepen the CRM functionality and enable report generation. We will combine payments from different invoices into single transactions reducing transaction costs for international payments. An alerts mechanism via email and notifications, real-time tracking of payment status.
