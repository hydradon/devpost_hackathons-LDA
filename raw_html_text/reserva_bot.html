Inspiration

From the exponential tecnologies we think that the best way to optimize the book shedules was with a bot

What it does

the bot connects to the reservamos api and take the user petition of the destination place, origin place, date of sheduling, and the way of travel (bus or plane), then the bot sends the request and display the trips that are avaible.

How we built it

We use Microsoft bot framework because of the familiarity of the language, node js for the server request and the main language javaScript

Challenges we ran into

The connection to the reservamos api, the display of the trips, for what we gona using.

What we learned

api request, bot develop and travel problems.
