Inspiration

Our inspiration is to assist maritime enforcement to stop illegal fishing.

What it does

The application aggregates fishing vessel data from various sources and stitches this information into individual records. It also works offline and updates when an internet connection becomes available.

How we built it

We built the back end using python along with the Django framework and Postgres database. The front end was built using electron, html, css, and javascript.

Challenges we ran into

Retrieving the information from multiple data sources and knitting them together into a cohesive dossier.

Accomplishments that we're proud of

We were able to knit multiple data sources together and integrate them into the framework to deliver a data payload to view vessel information.

What we learned

We learned to obtain specific and relevant vessel information from multiple data sources and streamline it to specific fields that provide precise, standardized vessel information.

What's next for KingTriton

Enable offline cache of vessel image data, adding more data sources, and finding strategic partners to promote our application for deployment on law enforcement vessels.
