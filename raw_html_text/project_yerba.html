• Project Yerba - Site title

Inspiration

To prevent fraud within the insurance industry.

What it does

Utilizing firebase on the back end, insurance companies are able to upload documents to a bucket (standardized form) that allows a consumer (recipient) to view their claim history. This will allow the consume to verify whether or not the claim is valid, and is able to raise flags for possible fraudulent claims.

How we built it

Quickly.

Challenges we ran into

We began building with python/django, switched the stack again to MEAN stack JavaScript, then took out the express and mongodb functionalities to switch to firebase.

Also, 2 way data binding with file storage was tough, we had to create an angular.js directory to properly bind data to the controller.

Accomplishments that we're proud of

We set up firebase, and got files to upload to the server. (Never used firebase before!) Also I was able to integrate angular.js into the application.

What we learned

We learned a lot... obviously firebase and cloud storage, but also planning and project management. We now understand how important it is to know the stack and implementation rather than switching 3 times.

What's next for Project_Yerba

We're building the site, updating our app, and making it pretty.
