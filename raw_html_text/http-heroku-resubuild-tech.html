Inspiration

As uWaterloo undergraduates, a majority of us spend a lot of time creating tailored resumes for specific jobs during the job search season. Many students we have talked to spend a lot of time trying to customize their resumes based on specific job requirements. We wanted to find a solution that would help our peers reduce the effort and time spent on doing so.

What it does

Resubuild creates custom resumes for students based on job requirements. The user can enter all their skills, work experiences, side projects etc. They can then copy-paste the same job fields found on WaterlooWorks (title, summary, responsibility, skills required), and the app creates a customized resume accordingly.

How we built it

We used Node.Js and a few dozen dependencies to built the backend (eg: passport, body-parser etc.). The frontend was created using JavaScript (handlebars template engine), Html5 and CSS (Bootstrap).

Challenges we ran into

-Login system did not want to work, database and app in general deployed on Heroku like 5 min ago

Accomplishments that we're proud of

We created a great looking web application in a short amount of time.

What we learned

Start with a plan early, and make sure you don't procrastinate.

What's next for http://heroku.resubuild.tech/

Create a WebScrapper to get information about ALL WaterlooWorks jobs and cut down the general job hunt time significantly.
