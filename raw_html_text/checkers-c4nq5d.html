Inspiration

We wanted something "simple" yet interactive and entertaining.

What it does

The idea was to have a player vs computer, but the player interaction did not work in time. Also, the user interface did not work as intended, as the merging of the engine with the GUI did not line up properly.

How we built it

One of us worked on the checkers engine, while the other worked on the user interface. It was not as optimal as we had liked, but we did our best to divide up the work.

Challenges we ran into

We struggled to do our own work while making it run as a whole.

Accomplishments that we're proud of

Getting the engine to work and the GUI to work separately.

What we learned

It's important to work on one project rather than trying to merge two.

What's next for Checkers

Making the GUI and engine starting from one program.
