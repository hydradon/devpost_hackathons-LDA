Inspiration

We are all immigrants in Miami, facing today's challenges :) Certain groups of patients are not getting medical attention because they don’t know how to traverse the healthcare system in the US, especially when they face cultural limitations such as language.

What it does

Simplifies the experience between patients and healthcare professionals to increase access and healthcare quality by overcoming the current limitations set by cultural barriers, specifically (and hopefully not limited to) language barriers.

Proposed solution is patient-centered, therefore the patient is the one that starts the interaction. The app would be used when a patient seeks medical attention, but feels limited by his communication skills.

Through a simple survey, the app should create a sort of quick reference profile that will be used to:

  • First, recommend the patient to visit a specific medical facility based on location, multi-lingual friendly, maybe healthcare provider, and current condition based somehow on the quick survey.
  • Second, intake information from the patient and create a translation to provide to the healthcare facility so they can select the most adequate healthcare professional.
  • Third, provide both the patient and the professional some additional communication tools (situation related keywords, etc.) that might help when they interact.

How we built it

We used React for an amazing front end experience, paired with Bootstrap for UI and Google API services such as Maps and Translation.

Challenges we ran into

Getting the nearby places from Maps Places API :D. Using 8base for data storage.

Accomplishments that we're proud of

Coming up with a solution that solves a real problem matched with a pretty straightforward interface. Great experience, first time working together as a team and it was a success!

What we learned

A great idea is more important than a great development. Connecting our apps with third party APIs, such as the amazing GCP!. Not always a complex backend is required, even more when using front end technologies like React.

What's next for MedLingos

We can complete the development with the adequate backend, some data storage for information persistence and data analysis and metrics. Can be extended to include other multicultural barriers, taking advantage of other technologies such as Text-to-Speech and Speech-To-Text, Visual recognition, etc.
