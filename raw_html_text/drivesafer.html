Inspiration

DriveSafe's current dispatching model doesn't allow for scaling/scheduling at a high volume of students.

What it does

DriveSafer automates ride scheduling for cars through use of Google Maps API. It mandates users have access to a McGill email, hence making it safer for both divers and riders. It alleviates pressure from the dispatcher, freeing the dispatch to focus on more important tasks. It would also naturally facilitate keeping track of various types of statistics and data.

How we built it

We were tempted to use new technologies that we were not comfortable to maximize our learning and have some fun. We used React.js for front-end development and Node.js and Express.js for backend.

Challenges we ran into

We were tempted to use new technologies that we were not comfortable to maximize our learning and have some fun. In addition, we were unable to utilize real-time location data from third-party applications.

Accomplishments that we're proud of

We accomplished a project in a language we were not familiar with. Our full-stack code is functional.

What we learned

It takes time to figure out how to use new technologies and actually use them as a part of a project. We learned a lot about how the basics of the new web-technologies worked.

What's next for DriveSafer

We are hoping to use real time gps location data from each car to improve the suggested scheduling. It would be also nice to use the stats that we will obtain to improve future scheduling suggestions.
