Problem

Branded event payment token and voucher system:

  • with prepaid vouchers for each participant
  • merchant accounts
  • payment system to send/receive tokens

What it does

Create a new token to be used as currency. Using “Stellar” as DLT payment network and “Blue Orion Smart Contract Library”. An ecosystem can be created with Event Organizer (asset issuer), user/attendee, merchants.

Technical Background

Blue Orion provides a smart contract library which enables voucher management. The Stellar network provides the decentralized payment system which enables transactions within 2-5 seconds.

Features

So, far we’ve developed the following value-added features, which are live on our site, blueorion.cc:

  • Non-Custodial Wallet : manage all your Stellar-based tokens, including with SMART tokens, Blue Orion’s very own crypto-asset, with History, Send
  • MyKeys : lets you monitor a KeyList, a set of public keys and their balance, from a static URL
  • Timed Transactions : schedule future payments on specific dates
  • Recurring Payments : send payments daily, weekly, or monthly
  • MyContacts : Assign names to public keys.
  • MultiSend : enables bulk token distribution and scheduling by CSV/XLSX upload
  • Vouchers : redeem a 5-digit code to get a wallet pre-funded with XLM & SMART tokens
  • Proof of Existence : validate integrity of intellectual property by a hash on Stellar blockchain Developers using Blue Orion blockchain services platform will be able to integrate the above features into their applications without spending valuable time on infrastructure development.

  • Ability for server to connect to data in QR codes
  • Ability for parties (users & merchants) to enter Access Codes to enter their wallet
  • KeyList for public monitoring off all addresses

How we built it

We are using Stellar protocol and Blue Orion Smart Contracts built on their API which supplies second layer infrastructure to make building for this use case simple and fast.

Challenges we ran into

We were trying to do a more generalized currency but we ran into an issue with fiat on-ramp/off-ramp.

Accomplishments that we're proud of

Getting a full-circle payment ecosystem up and running.

What we learned

Creating fiat on-ramps/off ramps complicates technology is a challenge.

What's next for Blue Orion

Our project’s mission is to provide a B2B platform for developers to use as second layer infrastructure to accelerate development timelines and to ease implementation of the Stellar network thanks to out-of-the-box API services designed for the most common payments use cases.

Blue Orion wants to provide everything developers need to build and deliver fully functional financial applications built on Stellar protocol.

Next, our objectives are:

  1. To develop the features listed above into a publicly available REST API
  2. To write proper documentation for developers to understand and use the APIs, including video tutorials
