Inspiration

Trying to help the first responders in the field streamline triaging, while helping command and control centers dispatch resources accordingly with real time data in disaster scenarios.

What it does

We ingest data from third party sources and first responders to give a detailed map of the current state of the disaster back to the first responder and a command center.

How we built it

Python and bottle on the backend, javascript and mapbox on the frontend.

Challenges we ran into

Third-Party platform integration.

Accomplishments that we're proud of

Our big data aggregation capability and the potential for our platform to become multi-faceted in the future.

What we learned

Mapbox, GeoJSON formatting, Geospatial context in programming.

What's next for Canary

Going fully live on Andriod and iOS. Using Google or Amazon platforms to run cloud based systems. Refining the frontend more to make our data analytics even more easily consumed by busy and stressed first responders. Improve collection algorithm. Possible API for third-party integration(i.e. active 911)
