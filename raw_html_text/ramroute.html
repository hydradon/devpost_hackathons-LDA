Inspiration

We brainstormed existing technologies/apps and problems. Then we decided to increase social interaction with our app.

What it does

It helps runners to generate random routes to explore the area as well as meeting the other runners. We are envisioning this app as a social running app where you can run and meet new people.

How we built it

We used Agile development methodologies during the hackathon. In this project, our team members learned backend NodeJS, MongoDB from scratch and used Google Map API & Capital One Nessie API.

Challenges we ran into

  1. We had problems with android SDK 23 permission requests, which prevent us from accessing mapping data. 2. Connecting Node backend to android.

Accomplishments that we're proud of

Trying new technologies. Reaching out of our comfort zone.

What we learned

  1. More info on android and how to use mapping GIS in mobile development.
  2. How everything comes together: Android, backend NodeJS server, MongoDB database

What's next for RamRoute

Get it to work properly. Add a feature to challenge friends and other people on popular running routes using a leaderboard system.
