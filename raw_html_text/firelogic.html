Inspiration

I work as an EMT and I want a way to be able to keep track of firefighters at a fire incident.

What it does

This application allows Fire Departments to register, add their firefighters and EMS to their department by creating accounts for each user. The Incident Commander can create an incident, track when firefighters enter and leave the building, and it allows EMS (Emergency Medical Services) or CERT (Community Emergency Response Team) to keep track of when firefighters have visited the rehab area, which firefighters are supposed to go to after they exit a fire to have their vital signs checked. The vital signs can be logged into the application, and Firefighters are able to access all data collected about them at all fire incidents that they have worked at.

How we built it

The front end was developed using bootstrap. The database was built in phpmyadmin and then the front end and the back end were connected.

Challenges we ran into

Lack of sleep. Lack of time. Difficulties when setting up the infrastructure to run the application.

Accomplishments that we're proud of

Full user account system is functional.

What we learned

Learned a lot about using bootstrap for the front end. Learned a lot about mysql stored procedures, and hashing & salting passwords.

What's next for FireLogic

Give firefighters the ability to get a print-out of their vital signs collected at fires to bring to a doctor or some other healthcare provider to show their history.
