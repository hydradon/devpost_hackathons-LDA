Inspiration

We wanted to build something that could help people make decisions based off their fitness data to make better transactions.

What it does

Queries the users purchase history and looks at fitness data to see the correlation between the two sets of data.

How I built it

We used Android Studio to build the actual app, and used Capital One's API to get simulated transaction history.

Challenges I ran into

Android Studio wasn't installed on 3 team members' computers which delayed development. During the project we encountered multiple bugs.

Accomplishments that I'm proud of

We were able to call the API and populate a table with the data that it returns.

What I learned

Have a plan of what you want to develop starting out. Keep it simple because there are going to be bugs. Make sure your team has all the tools they need to be successful. Ask for help early on.

What's next for FitTransactions

We are working to integrate actual fitness data using Android wearables as the data is currently simulated.
