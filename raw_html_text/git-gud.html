Inspiration

We wanted to control our repo with voice commands.

What it does

Allows for hand free repo operation through voice commands. Helps developers relax and evaluate their repo status through audio context.

How we built it

We developed a flask server to act as a restricted shell to process our commands. Commands were parsed via AWS Lambda functions getting input from Alexa.

Challenges I ran into

Limited time with wifi connection. Flawed documentation for the Alexa product combined with it being a new experience

Accomplishments that we're proud of

Alexa works with our spoken commands and can respond to them through voice Our api server acts as a restricted shell to ensure overall server safety

What we learned

We learned how to use flask, how to debug Alexa, and how to set up a hosted server.

What's next for Git Gud

We would like to add parameterized features, which means more dynamic code in Alexa integration.
