Inspiration

We wanted to crowdsource event discovery, by creating a social map. We let people find events around them and make plans by chatting with other people going there.

What it does

We created a social map, to find events and chat with people going there. Find the best food, music, and entertainment in your area. Discuss with your friends and other people attending or who are already there, all from our website!

How we built it

We built the web app using HTML and CSS. We integrated the Google Maps API to create the structure for our social map. The Cisco Spark API was implemented to allow us to create chat rooms between event patrons.We used firebase to store all event data and user information. We used javascript to push and pull event information from the database and implement the Cisco and Google Maps API.

Challenges we ran into

We ran into a frustrating issue with linking the infowindows to the actual location markers in our google maps implementation. The last infowindow was being linked to each marker, instead of having a seperate infowindow to each marker. We fixed it , however.

Accomplishments that we're proud of

Proud of the design of the final product. We're proud that we didn't have to hardcode any data.

What we learned

How to use Cisco's Spark API to integrate group chat functionality into web applications.

What's next for hang|time

We plan to create filters to allow people to filter through available events.
