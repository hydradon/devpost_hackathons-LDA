What it does

Solves the elementary dilemma of not being able to decide what to order.

How we built it

We kicked off our journey with a brainstorm session. We came up with different ideas and concepts together. A couple of hours and several group decisions later, we made the first basic structure of the app on paper. We then divided the work into parts like collecting data, working of the preliminary design, setting up the server and programming the back-end.

We collected data (including name, spiciness, image etc.) and prepared a database of approximately 100+ food items from 3 different cuisines.

We decided to build Hemu on PHP, HTML, CSS, Javascript and mySQL so that it could work on any phone (Android/iOS) without much effort. The main library used in Hemu is provided by "Materialize CSS - A modern responsive front-end framework based on Material Design".

(Some of the components that we used are code-pens, and stack-overflow questions. )
