Inspiration

Being avid members of the gaming and live streaming community, we were greatly intrigued when we learned that Standard Library's Autocode beta had Twitch integration. We thought of the potential of building tools for game developers to create games that involve both the streamers and the spectators, bringing everybody together in one.

What it does

Twitch Game Tools is a proof of concept Twitch Chat command query API built in Node.js and Autocode. It is built to allow game developers to easily implement a tool that will connect twitch streamers to their audience through the game.

How I built it

It was built using Autocode to handle commands and queries, the Twitch API to handle messages and commands, and Node.js and Phaser 3 to create the back end and demo game.

Challenges I ran into

  • No twitch chat support through Autocode
  • Difficulty with integrating all components of the project
  • Learning Http requests in JS

Accomplishments that I'm proud of

We are proud of completing the game and having a fully functional prototype. We are also proud of our versatility and productivity in our teamwork, allowing us to all work on the project at once and create a fairly polished final product.

What I learned

Our team learned the functionality behind Autocode, as well as how to deploy and create APIs using it. We also learned how to integrate the Twitch API into different projects.

What's next for Twitch Game Tools

Assuming a wider breadth of commands slowly being integrated into the new Autocode ecosystem, we hope to move all of the twitch chat interactions into Autocode, allowing us to quickly prototype and deliver a full API that provides all the tools we wanted.

We also hope to implement Google Cloud Natural Language Processing to make interactions with the game more natural and seamless, rather than commands.
