• red maple project

Inspiration

Most of the apps on the market depend a lot upon data entry by their audience. We wanted to create something that is not user-demanding but it provides them a better standard of consumer choices.

What it does

It analyses customer bank account report and checks the companies they are shopping at to give them a better idea of how much of their money is going into helping companies that are working towards being environmentally sustainable.

How we built it

We started building it with html files with css and we used node.js for testing on localhost and then we used the free domain from Domain.com to build the website. We used app engine from google cloud to host it and we used the Places API from google cloud to provide customers with more info on the companies and we used cloud sql also from the google cloud console for the backend where we used data from the Dow Jones Sustainability Index.

Challenges we ran into

Propagating nameservers from Domain.com took a long time and we barely got it done in time. Another problem was using library chart.js which is not responsive by default and it took us several hours to find the options that could scale properly. The biggest challenge was the time constraint.

Accomplishments that we're proud of

Coming up with a pretty great idea that would have a real-world impact.

What we learned

We learned a lot about the google cloud console, web dev and filtering data in general.

What's next for https://github.com/shahrzad88/red-maple-project

Just working on the project in the future until it has a stable release with polished UI and UX and hopefully working on an Android App to accompany the website and keeping it open source so its accessible to people.
