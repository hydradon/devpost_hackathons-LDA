Inspiration

  • A group member experienced the difficulties of communicating with EMS
  • Two other group members have training in emergency first aid, and realize the long procedure in calling EMS.

What it does

  • For those with medical conditions (e.g. angina), if they were to have a reaction(heart attack), it would only be a simple touch away from calling for help rather than the long process involved with explaining the details in a call.

How we built it

  • Using marvel, we developed a model of the ui for our project

- Used pygame and python to create the submission.

Challenges we ran into

  • Time constraints - unable to implement all the features we wanted to include in the project (a database where we can store all those who requested for help and rank them in order of importance similar to the process right now). For the purpose of the hackathon, we were going to add a sms api but we had run out of time.
  • Also unable to turn the pygame into the app, as we had troubles installing the necessary softwares.

Accomplishments that we're proud of

  • Persistence in the face of dev's block, went through many ideas and decided upon a reasonable creation we were satisfied with.

What we learned

  • Importance of having a good user interface

What's next for Quick Meds

  • partnering with the peel emergency services to connect to their database and send the requests of help
  • developing the app using python kivy
