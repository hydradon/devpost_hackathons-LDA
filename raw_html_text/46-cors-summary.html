Inspiration

NUS students are required to bid for their modules prior to the start of each semester. However, it is inconvenient for students to manually search through the CORS archive to gather information on past lowest successful bidding points.

What it does

It presents the lowest successful bidding points history in a visually friendly and convenient interface.

How we built it

Through the use of HTML,CSS,Javascript and Highcharts.

Challenges we ran into

Unable to integrate a database into our application within the given time frame.

Accomplishments that we're proud of

It provides a good conceptual model of what the actual application could be if it supports database integration.

What we learned

A deeper understanding of how various tools can come together in order to create a useful application.

What's next for 046 - CORS Summary

Incorporating database integration and improvements to the user interface (allowing users to type their filters, such as modules, as there will be a large amount of modules).
