Inspiration

We wanted to be able to communicate songs easily and handily. No better way to do this than through email. We also wanted a tool to summarize links emailed to correspondents. After all, most people have very short attention spans when it comes to emails, and an email sender must do everything they can to command an audience's attention.

What it does

Part one of our integration is able to send people song links simply by searching it through email. Quite helpful for discussing music. Part two creates a short, intelligently created summary of a text article, so that the reader may get a quick glance at the article to see if it's relevant to them or not.

How we built it

We used the MixMax integration API, the Spotify API, and the summarize API.

Challenges we ran into

Many issues with embedding iframes into emails. Turns out gmail tries very hard to protect users from phishing scams.

Accomplishments that we're proud of

Creating some custom CSS.

What we learned

We learned a lot about node.js, making and utilizing APIs, server side code. All great stuff.

What's next for MixMax Spotify + Summarizer

MixMax Spotify can do a lot more in terms of music integration -- Spotify is just the beginning. The summary component utilizes someone else's API for now, but we can use modern MLP models to create far better text feature renderings that people respond to. We can even use MixMax email tracking API to see how people interact with various models of summary and use reinforcement learning in this way.
