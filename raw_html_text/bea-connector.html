Inspiration

Working with BeA on a daily basis is a real struggle for german lawyers. It's an unreliable system that only creates problems for the daily work.

What it does

It connects to the BeA platform and receives the number of unread messeages.

How we built it

We used postman to figure out the api flow and built a rough prototype in node.js to perform the calls.

Challenges we ran into

A really bad api documentation, with lots of falsy information.

Accomplishments that we're proud of

Performing the login flow with BeA

What's next for Bea Connector

Building a PlugIn for systems that lawyers are already used to.
