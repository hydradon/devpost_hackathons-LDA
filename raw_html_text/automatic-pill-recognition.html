Inspiration

I'm the creator of PillSync.com and for almost 10 years, the site has helped thousands of users identify pills via their phone camera. As more people are taking more pills, I needed to find a faster way to ID them (in real world conditions). The first step to doing so is to segment the pill via Object Detection. I've tried all the Cloud AI like Google Vision and IBM but all of them cannot reliably detect Pills. These algorithm have only been trained to detect common objects like Balls, Eggs, Coins, etc. During my research, I quickly realize that I needed to custom train my model using tools like Tensorflow. However, this would require some time to master the learning curve. This was what led me to discover Azure AI Custom Vision.

What it does

At the most basic functionality, the custom Object Detection can now Segment the Pill and return the most common Pill Shapes of ROUND, OVAL, and CAPSULE. This Prediction API is being used daily to crop pills at a high success rate above 95%+.

How I built it

I went through my collection of pill photos and organized them in buckets of Round, Oval, and Capsule. Using the portal at CustomVision.ai, I uploaded each bucket and began to Tag the Pill in each Image. The UI assisted me in locating the desired Bounding Box quickly. Only occasionally did I have to manually drag the box. Initially, I tested with 50 Round pills and used Fast Training to see how well it works. Using the Quick Test mode, I was able to upload pills and input URLs of recent pills to test the success rate. For Iteration 1, I did notice many errors for cases where no samples were used in training. So I started to upload all the pill images that didn't work to the training set and tag them. After a few hundreds more pill photos and 6 Iterations later, the Precision % increased significantly using only the Fast Training that took only minutes! I did try the Advanced Training once and it took longer (less than 1 hr) but not sure if it's that much better.

Once I was satisfied with the API, I created an automatic script to crop the pills using the Crop Vertices returned by the Prediction API. I've run hundreds of recent pills and found only a few errors.

Challenges I ran into and What I learned

Machine Learning training requires a quality training data set. This means finding all the edge cases. For Images, it's a little harder because you have to See the Photo. The most labor intensive process was using Human Vision to locate quality training Images and Tag Them. The good news is that it doesn't have to be Thousands of Samples. In my case, I only needed a few hundreds.

Accomplishments that I'm proud of

I was able to find a relatively small selection of pills in my large collection to establish a functional custom Prediction. I'm very happy with the automated script to crop pills everyday as this has been a huge problem.

What's next for Automatic Pill Recognition

I will continue to test and iterate more Pills as errors arise. Now that each Pill is Zoomed In, I'd like to try the next step in the Heuristics of Image Classification of Pill Scoring 1,2,3,4. This will help narrow down the possibilities. Additionally, I've already started working on a Pill Labeling and Retrieval system to quickly Label more unique Traits and return the corresponding Pill Photos to feed the Training Set.
