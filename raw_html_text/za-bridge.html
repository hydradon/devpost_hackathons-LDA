Inspiration

Programmers should follow one simple principle: DRY (Don’t Repeat Yourself). Then why our Finance team should add invoices twice? #uipath #automation #saga #xero

What it does

It takes data from Xero using an API, we decore data to match Saga, then we send data to RPA which will add / update data to Saga.

How we built it

We created a custom API using Symfony and we are using our robot to request for all invoices (updated / created).

Challenges we ran into

UiPath is a new technology for all of us, so we spent some time trying understand its components. Also, it was really challenging for all of us.

Accomplishments that we're proud of

We have a fully functional application. :D

What we learned

We can use RPAs to make a lot of things work with human interaction.

What's next for Za Bridge

We see a bright future for this application! We can make a lot of improvements and our invoices workflow can be improved.
