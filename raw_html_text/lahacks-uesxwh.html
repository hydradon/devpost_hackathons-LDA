Inspiration

This is an education application to get historic sites, or landmark around users' current location.

What it does

Based on User's location, HistoryGeek introduces historic sites, museums or landmark around user. User can click the marker from the map, read short description, picture that other user uploaded, or read their comment. And user can share those sites with their friends or family through SMS message.

How we built it

We used google map to show user locations of sites, and foursquare api to grab historic landmark around user, and also pictures and comment. And with the Twilio, user can share their experience with their friends and family.

Challenges we ran into

Integrating multiple APIs together in limited time. It was challenging that we have to understand how does Google, Twilio, and Foursquare Apis are working, and put together so it works as purpose of our app.

Accomplishments that we're proud of

We are proud of that we were able to make our application works as what we planned in the beginning of the cohort.

What we learned

How to use multiple Api include Google, Twilio, Foursquare. Working with location based application. Building meaningful application within given time span.
