Inspiration

We were inspired by the overwhelming amount of poster and notices on college campuses. The best method for finding out what's happening was to see a poster, take a photo, and remember it a few days later. The constant bombarding of information from hundreds of different sources led us to create one central place for sharing events. It's 2016 - there's no need to continue to share, memorize, and try to find events on paper.
Facebook events only reach invited friends, there's no way to let the whole campus/area know about an event.

What it does

Allows users and hosts to create events (anonymously if they wish), specifying a place on the map, time, and details. Users can up vote or down vote the events to prevent against spam. You can comment on events and search. You can easily scroll through the days to see whats happening from today to next year.

How we built it

We used Xcode and Swift to create the mobile front-end, and the backend using a LAMP stack.

Challenges we ran into

Getting autolayout to work across devices and orientations.

Accomplishments that we're proud of

We got it demonstrable! The event creation and viewing, voting, commenting, and searching is functional.

What we learned

We got much more effective using PHP and SQL.

What's next for Mappy

Integration with Facebook events, add events to calendar, save events and get notified, more fields to event creation (times, host, photos, links, etc.)
