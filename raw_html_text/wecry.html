Inspiration

We thought about tackling Disney's hack that was focused on entertainment. When we were brainstorming we thought about what we do when we get home and they said cry, so we ran with it.

What it does

it tells you where you can go cry.

How we built it

html based with using the google API for directions and Javascript/CSS to make it look pretty.

Challenges we ran into

We wanted to use node.js and MongoDB, spent almost all day trying to get it up and running and realized that we didnt know how, so we produced a MVP so that we had some proof that we were here.

Accomplishments that we're proud of

We liked the idea and we are proud that we tried to go outside of our comfort zones, trying to learn new frameworks and skills that we may not have pursued otherwise, we may have not done it right but we know how to do it wrong.

What we learned

We learned that next time we need to learn the basis of what we will be working with before we build the app, diving in and trying to just make it work without understanding the basics was a bad idea. We also learned that we should develop a design process more concretely before starting.

What's next for WeCry

We would like to get together some weekend and add the functionality that we failed to use, mainly MongoDB, node.js, saving session variables, getting a more accurate location, reserving cry spots, organization of group cry sessions, and some general formatting things that seemed kind of off. then who knows, maybe we will put it up on a domain and some tech kids can have some fun with it.
