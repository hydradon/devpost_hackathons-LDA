Inspiration

We wanted to build a charitable fitness app that would hold users accountable for their stated fitness routine.

What it does

Users can configure their desired number of sets & repetitions/set for each strength-building exercise (i.e. squats, lunges, and glute bridges). SpotterBot keeps count of the user's completed sets and tracks the weekly progress for each exercise. If users do not meet their goals for the day, SpotterBot accesses the user's wallet to send an automated donation to the user's favorite charity.

How we built it

We used android's in-built sensors and capital one's API.

Challenges we ran into

Our time was greatly divided between feature implementation and design choice improvement.

Accomplishments include

Accomplishments included consistent sensor feedback and user customization.

What we learned

We learned a great deal about sensor detection and material design user interfaces.

What's next for SpotterBot

Increased customization, more exercises, and integration with the outside environment.
