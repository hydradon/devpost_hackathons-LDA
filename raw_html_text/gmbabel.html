Inspiration

Game masters who have a hard time not making all their non-player characters sound like pirates

What it does

Allows a GM to create characters with different voices, and speak as them through text.

How we built it

We used an Agile methodology to develop our app. We did our development in Visual Studio, and used Amazon Polly for text-to-speech. Our app is written in C# .NET and uses WPF for it's UI.

Challenges we ran into

None of us had actually worked with AWS Machine Learning before, so we had to overcome a learning curve to properly call Polly and use authentication correctly.

Accomplishments that we're proud of

It works! We can make our computers talk. Using many best-practices like object oriented programming and agile development.

What we learned

A lot about AWS, wpf, C#, web-services, and much more!

What's next for gmBabel

Continue along our road-map, adding new features. Then we'll do user-acceptance testing (make our GMs use it) and, if all goes well, release it to the wide world!
