Inspiration:

Incidents from personal life experience, where my parents, where, not able to access the technologies to get medical assistance. So we decided to build an interactive model which communicates to those who lack the ability to access technology to get medical assistance.

What it does

It provides a solution where in users can interact with the virtual digital nurse using voice interaction to get instant medical advice.

How we built it

Regarding architecture our application comprises of 3 components -

  1. Backend component: "apimedic" - API for getting diagnosis information
  2. Middleware component: "Python" - for creating a client-server model to interact with the backend and the frontend components.
  3. Frontend component: "AWS" - for developing the Alexa skill for providing voice interaction with the users.

Challenges we ran into

Making apimedic to communicate with Python.

Accomplishments that we're proud of

We have successfully developed a working prototype of an application which will prove extremely useful to the society.

What we learned

We learned a trending area of application development.

What's next for Insta Nurse - Alexa Skill for Medical Assistance.

To take it to the next level where it can be more interactive and provide more in-depth diagnosis and also suggest recommended physicians.
