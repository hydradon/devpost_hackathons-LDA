Inspiration

  • There is no systematic way to find the best relevant precedents to use as a template for a new deal.
  • Current methods are inefficient and do not always yield the best template.

What it does

  • Upload old deals and automatically create a database organized by key features.
  • Find the best precedent to use as a template based on the type of deal you are drafting.
  • Don’t rely on memory or contacts to find templates. Find the best precedent with a few clicks.

How I built it

  • Build front-end based on Bootstrap and back-end using CodeIgniter framework.
  • Finding the best precedent is using HP API, which returns most related documents we uploaded based on keywords or categories user select.

What's next for Legal Template Machine

  • Add flexibility to pull data into a template by section rather than using entire documents.
  • Add easy access to original deals that you are using in a template
