Inspiration

We are excited about Aave's flash loan protocol and the potential it has on the de-fi space.

What it does

Ficient allows for developers to create arbitrage circuits across decentralized exchanges while leveraging flash loans to create value in a risk less manner.

How We built it

Our front-end is built with React while the backend is written in node and we used solidity smart contracts with the Aave protocol

Challenges I ran into

Front-end: Interacting with the smart contract through web3 library on the react app was challenging as some of the web3 functions were not included in the base node modules.

Back-end: Breaking changes to Solidity depending on version and compiler. Generating factory contracts and lack of error messages.

Flash loans are new so documentation is sparse so we were paving our own way.

Accomplishments that I'm proud of

Implementing flash loan functionality while connecting the front-end to the back-end and solidity functionality.

What I learned

A lot about the Aave protocol and interacting with decentralized exchanges and also the concept of a flash loan and how we can use it in de-fi.

What's next for Ficient

Allow for other use cases besides just arbitrage and support more decentralized exchanges.
