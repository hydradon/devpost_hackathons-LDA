AdSee eats bad ads

Adsee is a little monster who lives in content land. Adsee likes good ads. Adsee hates bad ads. He hates bad ads so bad he wants to eat them all up.

Allows users to control what ads are displayed for them

Users are not blocking the ads because they do not want to see it. They block it because they are annoying and they take up load time etc. Whether or not an adblocker is present Adsee sees the ads. Using the Adsee dashboard the user can decide what ads to blacklist, campaigns that will be gone forever, or which to whitelist, will be excluded by the ad blocker. Adsee does that because it is speaking to the ad server instead of the user device. Adsee is esentially a little embed code that connects to an API which will store the user's preference, feedback, about the ads. Advertisers will benefit from this information because it is very specific and gives them new ways to evaluate their campaign creatives. Users will benefit from Adsee because they will be in control of what ads are displayed for them.

Mobile version

Currently Adsee only works for the desktop browsers and the live demo assumes no adblocking is present. The next step would be to include the blocked ad dashboard and the mobile version.
