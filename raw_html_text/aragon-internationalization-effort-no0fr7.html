Inspiration

Our team met in Paris EthCC19. We were all planning to use Aragon as a tool box for cooperative projects. However we agree that English was an obvious adoption barrier for locals from Barcelona and Madrid. This hackathon has been the perfect excuse to create a team and find out how to provide a collaborative and flexible translation framework for Aragon OS.

What it does

Our Aragon fork automatically detects the language of your browser and displays Aragon platform according to your language preferences. Furthermore, we set up a collaborative project for continuous translations so everyone can contribute translating Aragon in any language,

How we built it

1- We forked Aragon project and we imported react i18-next, a powerful internationalization framework. 2- We set a Weblate project in 3 languages and connected the github repository of our fork. 3- We labelled all the Aragon content through i18 scanner react function we extracted all the content to a json file. 4- From Weblate we pull json contents file ready to translate in any language.

Challenges we ran into

A day before hackaton an Aragon update broked the dependencies, so we had to use a previous version of Aragon. Aragon code is quite a monster with thousand heads. Learn penguin language in 24h.

Accomplishments that we're proud of

Recovery of Aragon previous version. The capacity to import react libraries and make the integration with Weblate. Automatic language detection through user's browser preferences. Learn penguin language in 24h.

What we learned

Work as a team. Find solutions under pressure. We get more familiar with Aragon architecture. We learn how to work with a collaborative translation platform

What's next for Aragon Internationalization Effort

Reach out people trhough SSNN campaing. Ask Aragon community for the adoption of our solution. Think the possible decentralized organization designs to manage the translation efforts.
