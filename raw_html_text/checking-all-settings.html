So far I have worked on many Uipath projects, most of them are used the Enterprise Framework. Default Enterprise Framwork template include InitAllSettings process and Config file. InitAllSettings can import key and value from "Settings", "Constants" and "Assets" of sheet in the config file, and all of them is kept in a Dictionary object. In most of the times, those configs need to be checked one by one before continuing process. So in this project, add in config file a "Checkings" sheet that has three types of configurable check and returns message column. Develop a new CheckAllSettings process that can be used to check all of the items in config valuable. If it is worked, can be used for future every UiPath project.
