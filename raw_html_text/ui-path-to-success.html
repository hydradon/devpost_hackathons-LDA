Inspiration

We began our journey with a few adventurous innovations yet concluded to one single point, to create a machine learning based UI Path data scrapper. The inspiration arrived from the idea of simplifying search for more sophisticated terms and products which usually have inaccurate search results.

What it does

The machine learning data scrapper can be used for a wide variety of purposes, whereby we focused upon narrowing down results of E-commerce businesses, namely Amazon and E-bay. The user inputs a certain product or category part of the aforementioned businesses and the scrapper seamlessly searches both websites and scraps the most relevant results to your search query and displays them on another external web-page.

How I built it

The product was constructed using a range of various languages such as XAML, Bootstrap, JavaScript all molded together using the UI Path Studio.

Challenges We ran into

There were several challenges with this project, especially the lack of experience all of our team members had with UI Path, a studio which none of us had ever used before. There were many issues in searching and scrapping web-pages where at times, certain web-pages or various search results would render hard coded data rather than the required results from user input.

Accomplishments that I'm proud of and things I've learned

Throughout the Hackathon, we achieved several goals including our accomplishment of learning majority of the UI Path studio during the event as well as gaining greater expertise in collaboration and improved application in error handling techniques.

What's next for UI Path to Success

To implement machine learning techniques to cluster items based on the latest and most cost effective deals which provide the greatest benefit for the user in terms of both economic cost and product quality.
