Inspiration : This app will reduce the stress of a patient to carry the prescription reports and files each time he visits the doctor.

What it does : Every time the patient visits the doctor, he has to save the given report/prescription given by the doctor into our app, so that the records can be maintained. In case of emergency we can use this data, to know the status of a person

How I built it : It's an android application, java is used to build the app in android studio.

Challenges I ran into : During the making of the app, our team ran into bunch of challenges. All of us being new to android app development, we ran into challenges like dynamically loading the list view, Tesserect OCR integration to the app, parsing the data we got from OCR, etc.

Accomplishments that I'm proud of : Apart from the many problems faced, we are happy to build a working health report manager app which a person can use to maintain a record of the prescriptions/report he got from the doctor.

What I learned : Learned Andriod app development and Tesserect OCR library integration. Apart from that learned debugging the code.

What's next for MedicalReport : Can be plugged in with the health apps, which generate data for heartbeat, blood pressure, etc., Set up alerts for medicines for certain health conditions, Set up alerts for family or emergency number base on analysis
