Inspiration

(1) Ease new Gatling users into using the stress tool (2) Saving many different configurations for future use (3) No way to gain experience from previous user / project (4) No real time graphical feedback when running

What it does

Provide visualization for editing configuration. Version control. Real time monitoring.

How I built it

Java Web application with single user and multi user version. Spring MVC+tomcat+gatling.

Challenges I ran into

(1) Integration within UI logic, backend and Gatling. (2) Integration between Application and Graphite

Accomplishments that I'm proud of

(1) Most goals reached. (2) Successful Integration Between Multiple Components (Web, Java, Server, Graphite) (3) Ability to Access Application from AWS.

What I learned

(1) Automating the packaging and deployment of applications with docker. (2) the latest Spring MVC and Spring security. (3) Visualize gatling output with Graphite. (4) Building GUI with HTML and JQuery

What's next for Gatling Visualization & Real Time Monitoring

(1) Integration with Gatling HAR Recorder (2) Beautify User Interface (3) Implement All Functionalities of User Interface (4) Provide Multi-User Access (5) Automated Installation with Docker
