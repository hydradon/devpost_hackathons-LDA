Inspiration

Reducing paper and allowing users to keep track of their finances easily.

What it does

Allows imaging of receipts and storage of data, and displays output in expandable list. Has an analytics page but it doesn't do anything.

How we built it

Android studio for the app. Python with flask and Microsoft's computer vision API for the server.

Challenges we ran into

None of us had any experience with Android Studio and it took a lot of effort to get to grips with. Started with Google's vision API which didn't work with our server.

Accomplishments that we're proud of

App takes photos and server returns data mostly successfully.

What we learned

Android dev is horrible.

What's next for Big Spender

Getting basic features working properly. Improving design. Adding analytic functionality.
