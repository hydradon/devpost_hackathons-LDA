Inspiration

Moving to a new city, whether to live or work or for immigration, can be challenging for students. TravelMatching aims to make the move easier with crowd-sourced reviews on the city, be it transportation system, living costs, and more.

What it does

TravelMatching lets users post reviews of cities for prospective students to gain more insight into their potential move. The posts will prompt users to share their own experiences moving to the city, which will be processed by our application to aggregate the score based on the quality of living. Further functionality to be added includes direct messaging options between accounts and a display of housing options.

How we built it

The front-end was built using Angular and TypeScript, while the back-end was made using Java and MySQL.

Challenges we ran into

Connecting the front-end and back-end was challenging. We did not end up finishing the back-end, as we had difficulties with connecting the back-end and front-end. In retrospect, we would have perhaps began using Spring Boot to simplify routing, but the learning curve was too great for a 24 hour period.

Accomplishments that we're proud of

We learned about how to do a full-stack web application.

What we learned

Managing our time within a 24 hour hackathon, as we are all beginners to NwHacks.

What's next for TravelMatching
