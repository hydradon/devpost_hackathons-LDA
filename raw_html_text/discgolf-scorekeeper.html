MESA 112

Inspiration

My Ultimate team "Cabritos Salvajes" and me, enjoy going to Rufino Tamayo to play Disc Golf, but keeping the score is always very time consuming between holes and at the end we're always adding points with everyone's phone calculators, so I thought there must be a way to make it easier and faster.

What it does

It helps keep the score for up to four players playing a disc golf course, summarizing the results in realtime.

How I built it

I used a lot of cool tools like codeanywhere, facebook sdk and canvas, zurb foundation, and javascript.

Challenges I ran into

gadamn javascript I couldn't use AWS containers, couldn't set up my free account domain.com coupons didn't work I needed a domain to be able to use https to be able to use facebook canvas, but it all got sorted out thanks to friendly advice and the Github Student Pack (shout-out to Luis Juan Sánchez Padilla!) I'm not as comfortable with javascript or web based development as I am with straightforward C++, C, or other nonweb languages, so I made a plaster of very inneficient code.

Accomplishments that I'm proud of

I finished the prototype, and it works!

What I learned

Working alone makes you deliver less features in a prototype since you can only do so much by yourself.

What's next for DiscGolf Scorekeeper

I'll pitch it to the Rufino Tamayo Disc Golf Association and make it into a full fledged mobile app with lots more features, like storing the results of each session in a database, generating "bogey" statistics, organizing events, and much more.
