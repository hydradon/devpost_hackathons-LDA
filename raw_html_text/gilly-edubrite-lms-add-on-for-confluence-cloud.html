Inspiration

Employee training is an old unfixed problem, and existing solutions aren't able to solve it for over two decades. The fundamental problem that needs to be fixed is - making training delivery simple and accessible and the learning content needs to be more engaging so more users are able to take it whenever they like it. Its our belief that integrating learning systems with platforms like Confluence, which is widely adopted in enterprises is the key to make training more accessible and effective.

What it does

EduBrite LMS is a SaaS learning management platform to build full learning solution for any business. Gilly integrates this LMS with Confluence so any user of confluence can enroll and take their training courses seamlessly, without having to go to a separate system. Gilly also provides macro to embed any course, quiz or survey created in LMS to be embedded in any page in Confluence. E.g. if you are building knowledgebase articles in Confluence, you can embed quizzes or feedback surveys in them. You can also embed courses built using videos, presentations, documents and external content like Vimeo and YouTube. Gilly's interface is fully mobile friendly and it delivers courses that are fully converted to HTML5 delivery format in LMS so they work on all mobile devices.

How I built it

We are using Atlassian Connect framework to build the integration with our SaaS platform, which is built using server side Java, MySQL, HTML5, jQuery and Bootstrap. We are using JWT authentication mechanism provided by the Connect framework.

Challenges I ran into

Well not many, though a few planned features in Gilly had to be trimmed or deferred to a future release until necessary APIs are available for Confluence cloud environment.

Accomplishments that I'm proud of

Gilly for Confluence cloud has built a one click signup and LMS microsite creation from within Universal Plugin Manager interface of Confluence, which is a nice way for businesses to get started with an integrated LMS platform, after they install Gilly.

What I learned

Gilly for cloud is a result of our learning and feedback about our existing add on for Confluence server. We received several inquiries about availability of Gilly for Confluence cloud which is why we are making it available for Confluence cloud. Since EduBrite LMS is SaaS (hosted), it was logical step to integrate with Confluence cloud.

What's next for Gilly - EduBrite LMS Add on for Confluence Cloud

New possibilities have opened up with this connect add-on, and we plan to integrate our learning management platform more tightly with Confluence during next 6 months. We will integrate full course authoring within Confluence to allow re-use of confluence pages while building courses, and add more reporting within Confluence.
