Inspiration

We were inspired by websites that provide random places to eat based on your location. We wanted to improve upon that idea by giving the user a healthy choice in addition to a place to eat .

What it does

We prompt the user for a location and will return fast food within walking distance nearby. The user can click on the difference places to find healthy, low calorie options from the different restaurants. Once the user selects a restaurants we provide walking directions through google maps to help further encourage healthy happens.

How we built it

We built this application using a mixture of APIs, including the USDA food database and Google Maps API. These are integrated into a dynamic website using PHP, HTML5/CSS, and Javascript. We host use Heroku as an online hosting service for the application.

Challenges we ran into

The team was very inexperienced with web technologies, with almost zero API and web hosting experience prior to this weekend. Additionally two out of three members have never been to a hackathon before.

Accomplishments that we're proud of

We were proud that we created what we felt was a completed product and were able to implement many of our stretch goals that we set forth on the first.

What we learned

We learned a lot about web technologies including Heroku, PHP and API usage.

What's next for Jared's Choice

We plan to leave the application on Heroku and want to make the website mobile friendly.
