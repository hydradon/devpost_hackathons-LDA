Inspiration: WeTime was developed with an aim to encourage and ease the process of people being spontaneous with free time without spending too much time looking for activities around and deciding if they are within budget, if they are open, or if the activities will meet their interests.

What it does: The web app collects information about users preferences, scours the web for activities that meet these preferences, and creates an itinerary with a detailed plan for the timeframe selected.

How we built it: The team used CSS, HTML, and Java to develop the web app.

Challenges we ran into: Learning CSS and HTML in a short time was difficult.

Accomplishments that we're proud of: Working well together and overcoming the aforementioned challenges.

What we learned: We learned a lot about web development and that any challenge can be solved by taking a little time off and getting back to the problem.

What's next for WeTime: We plan to introduce more user filters, location search, and an advanced search options.
