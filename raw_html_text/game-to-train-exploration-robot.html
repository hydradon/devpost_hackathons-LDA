Inspiration

This skill is modeled on a Mars probe. The skill development was based on the idea that the relationship between the player and the robot in this skill is similar to the distance between a person on Earth and the Mars probe.

What it does

You can use 4 commands.

Explore: Send the robot to the exploration. It takes as much time as real time to return from the exploration. When returning from the exploration, the robot will discover new knowledge. Also, robots sometimes pick up “rare metal” items.

Remodel: remodel the robot. it requires “rare metal” to remodel. (The number of rare metals required for remodeling varies depending on the level.) Remodeling will shorten the time required for exploration.

Status Check: you can check the robot status. You can check the current level of the robot and the number of "rare metal" possessed. In addition, recently discovered knowledge is displayed on screen devices.

Purchasing： You can purchase in-skill purchasing content "rare metal 5 pieces pack".

How I built it

Built by Alexa Developer Console, AWS lambda, DynamoDB, S3. Deployed with ASK-CLI using Visual Studio Code.

Challenges I ran into

I had a hard time building UI and UX using APL.

Accomplishments that I'm proud of

By using APL standard commands effectively, I realized interactive expressions.

What I learned

I learned that audio content can also have purchasing elements like smartphone apps. The Japanese audio content market is still developing. In the future, I wanted to develop in-skill purchasing skills and expand the audio content market.

What's next for Game to train exploration robot

Currently only Japanese is supported, so I would like to support English as well.
