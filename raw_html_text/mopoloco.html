Inspiration

The places that internet search engines recommend do not always correlate with the best places to see and go, our inspiration was the ability to see the places people actually go and be able to search which ones are most popular.

What it does

Our website geolocates users, stores their location and reverses geocodes to find their address. This data auto-populates a database stored in the Google-cloud. This data would eventually be searchable to find the most popular restaurants, places or locations by popularity.

How we built it

We tried using HERE's and Algoria's APIs to ingrate with our website.

Challenges we ran into

Not everything works because we had difficulty integrating and bending the APIs to do what we needed.

Accomplishments that we're proud of

We tried and we came up with an awesome idea... if it worked all the way.

What we learned

We gained a lot of experience using several different kinds of APIs and how to integrate them into web development. And we watched a lot of youtube how-to videos on Javascript, and RESTfull APIs.

What's next for MoPoLoco

Creating the integrated search engine using Algoria's API.
