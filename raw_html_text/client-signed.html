Inspiration

I wanted a way to run an MMO game without needing a player database. I also wanted to simulate a remote server by using web workers in the browser.

What it does

It simulates a remote server by starting a web worker and adapting web worker message posting into a web socket API. There are two clients that connect to it. When the clients does something "significant", which in this game is the act of picking up a coin, the server digitally signs a JWT back to the client with a private key. The next time the client reconnects to the server, it sends the JWT, the server verifies it, and instantiates the player into the world with the correct amount of coins and position.

How I built it

JS, browserify, and a crypto library.

Challenges I ran into

Webpack was working last week but wasn't today. No idea. That's web development for you. Also, 90% of my time was spent building a simple MMO using a communication technology that I don't think many people would want to use. Finally, this is a fairly technical concept so I worry if people will understand it without more explanation than I have time for.

Accomplishments that I'm proud of

I was able to proof out the concept within the time frame, and I will be using it in a future real project.

What I learned

That this is possible. And that hackathons aren't really my thing.

What's next for client-signed
