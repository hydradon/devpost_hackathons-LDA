Inspiration

To build a software to find lost documents/properties around the globe, which could be used by countries, cities, big organizations, and even institutions.

What it does

Finds lost and found documents.

Using mobile-friendly UI it gets two types of inputs: lost and found. All the data is stored on OneDB where it can be managed and searched for a match. This job could be done by a human or a computer, either work. Finally, if matched, a person who lost a document will be contacted and shared contacts of a person who found docs. Since my platform is web-based, it is fast and scalable. Literally, the whole world can use it. The only limit is the size of the database :D

P.S. Only car plate is available for test.

How I built it

HTML5 / CSS3 / jQuery / JavaScript + OneDB

Challenges I ran into

What's next for Global "Lost & Found"

Finish different types of docs for the user side app. Then, the development of an admin panel for searching, matching, and reaching out to people by SMS automatically. Long-term: try to integrate AI / Machine Learning for my platform to help the system recognize different things and categorize them.
