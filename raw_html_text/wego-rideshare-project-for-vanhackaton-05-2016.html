Inspiration

The Challenge:

My startup idea (WeGo Ridesharing) revolves around building an Expedia for everyday commute. To help people make an informed decision about their commute, we compares taxis, carpooling, car sharing, car renting services. We also enable ridesharing in each of these options to further lower the fare for people. 

I plan to start with taxis only. Build an MVP that includes the feature to match people commuting on taxis between same start and end points.

What it does

The people can register their Routes, and then it´s possible to share with other people, and the user still can choose the best price for the Ride.

How I built it

.Net, Javascript, SQL. Google Maps API

Challenges I ran into

The great challenge is to code the real necessity of the challenge.

Accomplishments that I'm proud of

I am very proud, in a short time-frame I had developed some simple, but great functionalities, focusing in the most important. With some more work, could be a great hub for sharing rides.

What I learned

To work with Google Maps API :)
