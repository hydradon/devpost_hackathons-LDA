TOM, short for Time Off Manager, is a bot application created to assist users in managing their personal time off directly in Slack.

Once TOM is authorized in the Slack team, and the Slack admin sets up the team, users, time off policies, and managers, the bot is added to the team. TOM does not need to be invited on a channel, because all the communication between the bot and the users is done solely via private messages. This way, TOM does not intrude in the daily activity of the team.

In terms of functionality, this first version of TOM, integrated solely with Slack, can help users with:

Taking sick days The user just has to write a PM to TOM, with his/her intent. TOM confirms the action, then notifies the team manager about the user’s status.

Taking vacation days In a private message, the user can express the intent to take personal time off. If the dates are not expressed in the first utterance, TOM asks for them, then confirms the timeframe and number of vacation days to be taken. Once the user submits the request, TOM sends a private message to the user’s manager, who can approve or reject the request. In real time, as soon as the manager takes any action on this request, TOM notifies the user with the response and subtracts the days off from the user’s total.

Seeing their scheduled time off Any user can request to see their scheduled time off, approved by the team manager. With a simple request to TOM, the bot displays a list of future PTO.

Having an overview of the team Every Monday, TOM sends a private message to all users with a list of people who are taking time off over the week. However, this list is available at all times, and is not limited to a week. Users can request to see who is taking PTO in any timeframe, by sending a private message to TOM.

Getting notifications when teammates are out of office Whenever a user requests a sick day, his or her teammates will be notified of this status, is order to clearly know who they can count on during the day.

Approving or rejecting time off requests Team managers enjoy the same functionalities as those of users, with one additional functionality - approving or rejecting time off requests directly in Slack. Whenever a user requests time off, the manager is notified and can approve or reject on the spot. If the timing is not right, the manager can ask to see an overview of the pending requests, and approve or reject them when the time is right.
