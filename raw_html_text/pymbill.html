Inspiration

We want to build an application that allows users to make some transaction on business, on the street and so on, Different from ATH Mobile that don't allows you make transactions on business.

What it does

It read a Qr Code that have all the information of the receiver and the amount of money you will pay.

How I built it

We built it starting with ATH Mobile Learning its API then followed through with android, were we found a nice third party app that communicates seemly with our app and reads QR codes. QR codes were chosen because their easily recognizable by devices and is wildly spread among them.

Challenges I ran into

We had problems with ath movil that made us adapt the code to later discover the sever was down. also some reasearch and process development did not worked accordingly.

Accomplishments that I'm proud of

The integration of a QR code with a transaction

What I learned

About the process of the transaction of ath movil and qr code implementation

What's next for Pymbill

Allow the user not justo to scan the Qr Code, but allow the user create a Qr Code with an specific amount of money that it will request to an other user.
