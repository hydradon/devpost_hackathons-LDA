Inspiration

The demo of Alona, where Roboy couldn't process the language because of a bad internet connection

What it does

It provides offline TTS and STT.

How we built it

Docker-Container for Kaldi Database, ROS Node for the integration for Roboy

Challenges we ran into

Dependency hell, bad microphone quality, HyperV does not support Microphones, No read-to-use environment to test Roboy

Accomplishments that we're proud of

Also created a VM for testing and building Roboy

What we learned

Using ROS is difficult, How to deal with complex distributed setups

What's next for RoboyOSR

Integrate in the offical Robo Repo
