What and how we did it :

We had a great brainstorming session as soon as we selected the UNhcr as our Use case for this Hackathon. We had performed a quick review and analysis on provided dataset and came up with the different classification of the Persona type and associated sentiment.

Technology Selection :

-Refugee Dataset -Agile, Brainstorming whiteboard discussions -S3 -AWS Comprehend -AWS Glue, DynamoDB, Lamda -AWS Quicksight -AWS API gateway to use analytics data to target specific user base on Facebook, twitter or any social media (Recommendation as an improvement/future suggestion)

How we built it

Used above-mentioned technology for Entities Detection, Key Phrase detection, sentiment extractor. Provide the Persona type to the API and the result would be the story/article with the most impact. It also provides the source, story, and the persona specific "donate" link.

Challenges we ran into:

-Data Quality -Understanding the persona type -AWS tool selection for text analytics
