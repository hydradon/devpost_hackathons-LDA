Inspiration

Lot of times we see some pets in the streets. Some might have the time and heart to help. Some might have the heart but not the time. This motivated Nagi to do the Rescue operations. However, for the rescue operations, Volunteers are not easy to come by. We wanted to build a cheapest Volunteer, Technology

What it does

Feature 1: Lost / Found Pets User takes a picture of a pet he/she finds in the road or any public place, Posts it in the app. The data is uploaded to the server and Social Media. It also fetches the lost pets that any user has posted and provides the information of those users who has lost their pets.

Feature 2: Location based Petcare services It provides details about nearest Vet and licensing place for dogs.

Additional Value Adds: Donations are integrated in to Paypal. Sharing to Social Media (Facebook, Twitter). Feedback system(Automates email).

How I built it Pet care

Backend: DynamoDB, Jersey REST Webservice hosted on Elasticbeanstalk webserver run on AWS EC2 instance. Frontend: Bootstrap

We also built an Android Application for the same.

Challenges I ran into Pet care

Since we built both Web and Android application from the scratch, We wanted the applications to be scalable, extendable and consistent in usages.

We resolved this problem by having the backend as a Webservice hosted in AWS elastic beanstalk server.

Accomplishments that I'm proud of Pet care

Hosted the whole scalable webservice in cloud. Built a complete end to end solution that is easy for Nagi to use.

What I learned

Technically, we learnt to integrate Paypal's API to applications; DynamoDB complex queries.
