Inspiration

To help shipping companies, exporters, importers, and insurance companies to receive information about the condition/integrity of the cargo

What it does

Use sensor to measure the cargo condition (temperature/humidity/vibrations/shocks/GPS location) and log the information Connect to the cloud and send alerts to the crew

How we built it

IoT - sensors Cloud infrastructure: data publishing/consumption/datastore/storage

Challenges we ran into

Lack of data break down from insurance claim and damaged good for market research purposes

Accomplishments that we're proud of

We generated ideas on the smart sensors for monitoring the cargo condition

What we learned

We learned about the maritime industry and the problems it is facing

What's next for SmartCargo

Need to implement a prototype of the device Refine the more relevant feature set for the device Create a standard API for the cloud to consume
