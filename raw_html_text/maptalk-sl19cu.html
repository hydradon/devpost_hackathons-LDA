Inspiration

We wanted to build an app for quality of life. It combines useful information about traffic, events around you, and also information about places you visit.

What it does

The user can see markers placed by other users, categorized by either a location, an event, or something in traffic. He can also comment on his own location with the aforementioned categories

How we built it

The backend server of the application is built with ruby on rails, with a PostGreSQL Server with RestFul API. The frontend, the apps are built with React Native, so we can use one code base to deploy to two platforms at once.

Challenges we ran into

We ran into a few challenges with the map implementation of the android part of the app.

Accomplishments that we're proud of

We're proud of the implementation of the backend server, which went pretty fast, so we had the time to focus on the frontend

What we learned

We learned a lot about how to use React Native to build for multiple platforms, and also alot about map implementation in this framework

What's next for MapTalk

Next up, there are a few modifications for the User Interface, for which we sadly run out of time for. After that, we'll be ready to deploy the app to the AppStores and hopefully start to gain a userbase. Maybe after that, there'll be ways to benefit from the app, for example with advertisements
