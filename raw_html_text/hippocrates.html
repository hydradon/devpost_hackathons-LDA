Inspiration

We wanted to speed up the communication process between patient and a doctor and also get rid of redundant tasks, such as having to go to see a doctor to get a prescription to medications. We also wanted to shorten the waiting times .

What it does

There is a chatbot which is the first communication interaface between the doctor and a patient. It is not a replacement for a doctor, but instead a tool which saves time.

How we built it

Firsly, we created front-end then back-end and then we created a communication channel between those two.

Challenges we ran into

Back-end development, React-Native mobile native application, Cross-Origin (CORS) issues.

Accomplishments that we're proud of

Fully working REST API written in Spring Boot, Maven and Java.

What we learned

We learned how to work with frameworks such as Spring Boot and Maven, how to build a fully working API.

What's next for Hippocrates

Finish android native app using React-Native Improve database Improve code quality Improve communication with the database via REST API
