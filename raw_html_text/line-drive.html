Inspiration

Frustration from waiting in long lines for food at baseball games.

What it does

Considers real-time data's line sizes to help users minimize wait time to get food, based of the user's food preferences and quadrant.

How we built it

We made a front end website that takes in input from the user and sends to a server. The server computes the input and compares with real time data using an algorithms and returns recommendations.

Challenges we ran into

Setting up a website, setting up a server, implementing algorithm in the server. Making the website work with the server and making the algorithm work with the server.

Accomplishments that we're proud of

We were able to set up a website, server, and create a working algorithm.

What we learned

The power of friendship and all the accomplishments mentioned above.

What's next for Line drive

Optimization, debugging, customer feedback.
