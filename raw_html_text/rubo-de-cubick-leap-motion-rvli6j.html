Inspiration

It came from personal interests between the team. We wanted to do something with the new leap motion tecnology and we decided to give it a try.

What it does

Using leap motion you have to solve a Rubick's Cube. Because of a lot of problems, solving the Cube is impossible. We have a demo that solve a Puzzle.

How we built it

We built it with Unity. We start the project by making it simpler than the final "product", instead of solving a Rubick's Cube let's try a easy puzzle. Once we had in mind and in paper all the stuff necessary to undersant minimally the leap motion we move on to the final step.

Challenges we ran into

  • Know the libraries of leap motion in Unity.
  • Get familiar with Unity
  • Solve a lot of problems due to our lack of expirience in this field.
  • Face magic results against all mental scenarios like random return values, impossible reacctions...

Accomplishments that we're proud of

  • We now know how to program the leap motion
  • We are quite familiar with Unity
  • Despite the final project does not quite have what we would like to, we are proud of the accomplishments.

What we learned

  • Javascript, Unity basics and how Leap Motion works.

What's next for Rubo de cubick - leap motion

  • Better graphics, more accuracy with the leap motion, better performance and get the code cleaner.
