Inspiration

Americas favorite past-time includes plenty of classic ball park traditions. Peanuts, Crackerjacks, and of course the Hot Dog Derby race. In the classic Hot dog Derby race, hot dog condiments (ketchup, mustard and onion) race around the bases to determine who is the favorite condiment.

What it does

This web based application is a multiplayer version of the Hot Dog Derby race. On the Jumbo-tron, an animation of the race characters will display. Spectators will use their mobile devices to join one of the teams and shake their phones to help their character win the race. The team whose players shake their phones the most will win the race.

How we built it

The application is build using .net MVC solution on the backend and JavaScript/HTML5/CSS on the front end. We implemented a custom REST API to allow the mobile devices and jumbo-tron to communicate with the web server.

Challenges we ran into

System architecture and hosting environment was a bit of a challenge. Since the gameplay involved mobile devices and a separate jumbo-tron animation, testing the application locally and elevating changes to a final hosting environemnt was a bit of a challenge.

Accomplishments that we're proud of

Being able to actually build a functional proof of concept for the mobile multiplayer race game in less than a weeks time!

What's next for Shake

Now that we have a working proof of concept, we want to refresh the graphics and update the system architecture so that the game can be played by an entire park full of spectators.
