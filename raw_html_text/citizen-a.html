Inspiration

As a team we have found that there is no real and no easy way to fill out a witness form. Additionally, we noticed that a lot of insurance claims go into dispute between the parties. Thus, we thought this application can both help the roads to be safer, and help the insurance companies to get a more accurate ruling.

What it does

It allows the user to submit their account of the collision. Along with their contact information and geo-location at the time.

How we built it

We built the application using Node.js to create a backend server and MongoDB to store the data from the users. As well, we used React.js to provide a smooth and intuitive UI.

Challenges we ran into

We tried to decentralize our application in such a manner that brings better security, and with it accountability for our application. Unfortunately, we did not have enough time to implement a solution given the constraints of the hackathon.

Accomplishments that we're proud of

This was our first hackathon, and we are very proud of the fact that we first of all survived, and second of all managed to create something that could be useful and effective in the real world.

What we learned

How to use new technologies that we never used before, different companies and new languages (along with workshops, and networking opportunities that allowed us to learn more about the sponsors)

What's next for Citizen A

Decentralize our application to ensure better security, and improve the accountability system. Additionally, add a map API that marks the location of the accident, that could encourage more people to provide a statement.
