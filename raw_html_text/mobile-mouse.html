Inspiration

We arrived at the hackathon ready to work(without an idea), pulled out our laptops and we realized that we forgot a mouse!

What it does

It is a program that lets your phone act as a mouse that connects to your PC.

How I built it

It is written in c#, android studio, unity, and github.

Challenges I ran into

  • 3d printing license not accepted
  • sdk for android studio was not up to date
  • we ran into problems with auto-desk inventor. we had previous experience using auto desk inventor but had getting acclimated to a newer version of auto desk inventor.

Accomplishments that I'm proud of

  • We are proud of the our concept program that we made to solve the problem of not always having a mouse on hand to conveniently use.
  • We also learned more about android studio. We learned how to implement android studio to talk to the cellphone so the computer could talk to the cellphone and receive the acceleration input.
  • I learned how to use slic3r!

What I learned

  • 3d printing (how to navigate autodesk inventor), slicer, g-codes, the printing itself -Android studio to unity -Unity to computer emulation -How mouse drivers work -

What's next for Mobile Mouse

  • Incorporating a mouse driver for windows/Linux/maxosx that takes the data we provide. -Wireless -scroll -position based, not rotation based -easy to install on all computers -To the app store!
