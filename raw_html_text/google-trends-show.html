Inspiration

We’re big fans of board games and local coop games. We took a game made by the YouTube channel Funhaus and automated the job of the moderator so everyone can play.

What it does

It gives the players a word and working on their own or with a group, they have to compete to find the more commonly searched related word.

How we built it

We made a client and server that could handle multiple concurrent games with varying amounts of players. We did this with HTML, js, and jsp. We used a python api to get the data from the Google Trends API

Challenges we ran into

Getting the Google Trend data was difficult, there is no publicly available official API and many unofficially ones were outdated and broken. Early in the morning, some strings would just disappear as they got modified. We eventually realized it was some weird character we got when reading from a file. We ended up just trimming it off.

Accomplishments that we're proud of

We made a game that’s fun and easy to pick up and play.

What we learned

We got better at working with clients and servers and using the tool that are required for that.

What's next for Google-Trends-Show

There are some features we had to leave out, like a more advanced team system, and scraping Reddit for currently popular terms related to different subjects.
