Inspiration

As Engineers we have a responsibility unto ourselves and our peers to make the code we write more readable and easy to understand. A huge part of this equation is project specific documents that are usually written in some form of a markdown format (md, rst, rmup, etc.).

The usual collaboration around these docs happens in a traditional code review style where a git patch is reviewed via different code review tools and the diff is annotated line by line. While line by line annotation works well for code, it is not as useful for writing paragraphs of text.

We really like Google docs and Office 365 and figured we can use the same collaboration protocol but for raw text with a WYSIWYG editor so that people can collaborate on engineering documents.
