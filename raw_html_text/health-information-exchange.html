Inspiration

When I went to shadow under doctors at a hospital, I realized how terrible the data infrastructure was. Because of the lack of medical information and patients would have to recall what procedures they had done on them for the past year. Medical mistakes are said to cause any where between 250,000 to 440,000 deaths per year in the U.S., and a primary cause of that misinformation, and the lack of medical information. I was this problem as saw this as a opportunity to help address the overlooked issue of inconsistencies with medical information.

What it does

Our application is able to integrate information from multiple EMRs or electronic medical records, which contain large databases. It then is able to place under one patient. This therefore alleviates the problem of a lack of medical information, and allows doctors to have more information to make better decisions.

How we built it

We built it using Django python framework. Then used the Google Cloud Platform store the data, so doctors and medical professionals can access it later.

Challenges we ran into

We ran into a huge problem integrating APIs, and the struggle of having only two people for such a large project.

Accomplishments that we're proud of

We are proud of how we are able to show the potential of this application for the future.

What we learned

We learned about API integration, mySQL and python Django web framework.

What's next for Health Information Exchange

We hope to continue to grow and show the potential of this project.
