Inspiration

What it does

Uses HTML5 webRTC to take a snapshot of the user for security, firebase for cloud hosting and Realtime Cloud Messaging to push mobile and webapp notifications to users on visiting.

How we built it

Built a website with HTML and routed user info through firebase CLI and Realtime Cloud Messaging API for messages over their network. Later implemention was going to include nexmo text messaging but we ran out of time.

Challenges we ran into

Connecting firebase to RCM, running node.js with non-node applications.Connecting firebase hosting and heroku together. Arduinos ran out quickly so we couldn't try a sensor for checkin.

Accomplishments that we're proud of

Getting the Push notifications to work and send, and setting up the framework for later server implementation.

What we learned

Tons of javascript, Firebase CLI

What's next for hackgsucheckin

Server implementation, texmo integration to text users. Perhaps Arduino sensor implementation so the webpage opens as the user steps in.
