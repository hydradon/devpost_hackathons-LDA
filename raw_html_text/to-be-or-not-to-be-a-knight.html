Inspiration

We wanted to build a fun old-fashioned text-based choose your own adventure game

What it does

Takes the player on an exciting and humorous journey as knight in training. The player chooses their name, can collect items, choose a companion, and if the player dies, it will be upload to the public deathboard using their characters name.

How we built it

We built it using python, sqlite, html/css, apache webserver. both the game and server side scripts are written in python.

Challenges we ran into

Time management

Accomplishments that we're proud of

That it's a fun and humorous game, and the network services work.

What we learned

CGI server scripting, and git.

What's next for To Be or Not To Be [A Knight]

We may continue expanding the game, and allowing people to download and play it when they want to.
