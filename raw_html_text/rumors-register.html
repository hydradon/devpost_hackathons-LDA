Inspiration - We have faced problems in the past because of rumors about diseases that weren't actually there.

What it does - It takes rumors from users that they are hearing around them and our doctors will filter them accordingly as they know better.

How we built it- We've used Surface framework with Twig templating engine. We've used php for the backend which routes the whole site so that everything remains organized. Simple but effective and fast user-interface allows to quickly filter the rumors from truth.

Challenges we ran into- Algorithm to sort out were to be built logically and that took a pinch bit extra effort.

Accomplishments that we're proud of- We've built good enough web application to handle the problem we have taken to overcome.

What we learned- Lots.

What's next for Rumors Register- Even simpler UI, automation of some processes.
