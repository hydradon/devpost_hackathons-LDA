Inspiration

When we heard about the travel game challenge, we wanted to make an app that encourages players to go out and explore, but also different from Pokemon Go in some way.

What it does

It is a game where players have to explore areas of the world they have not explored before to uncover treasures and loot, and survive against hunger, thirst and monsters.

How we built it

We used Google Maps Android SDK to display the area around the player. The sdk has extensive capabilities to draw images and shapes onto the map, which let us implement a "fog of war" around the player, and also draw treasures, monsters and other game objects onto the map. The game uses Firebase Database to store game data, and a Java server is used to create game objects and manage other game data.

Challenges we ran into

Although Google Maps SDK is powerful, it wasn't designed for gaming. We had to try different ways of drawing to get the results that we wanted. Organizing the development process and making sure we're on the same page was also challenging. All of us ran into various problems that slowed us down, but we somehow managed it in the end.
