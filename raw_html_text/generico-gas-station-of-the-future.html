Inspiration

We were heavily inspired by the emerging use of mobile technologies to streamline the process of transactions in the gas station and associated convenience stores.

What it does

First, it creates a quick scan / id based pump location system, to determine the store inventory and service listing for client browsing and purchasing. Second, the system automates the process of gas payment and convenience purchases through the use of semi-autonomous shelving robots, receiving shopping lists virtually, and collecting items for the client to pick up at a kiosk.

How we built it

We built a preliminary system of storage in Java, but quickly decided to discard that prototype to create a more web friendly Javascript and HTML solution.

Challenges we ran into

Learning integral components of the Javascript language's functionality. Mocking up data in spreadsheets to show our concept.

Accomplishments that we're proud of

Parsing CSVs in Javascript, and generating web page content in a legible format.

What we learned

Using a language we are unfamiliar with (although probably a bad idea in hackathons) and making progress within this environment.

What's next for "Generico" Gas Station of the Future

We would have like to have done a mock-up of the internal store structure, with the shelving bots collecting a client's items and delivering them to a kiosk for pickup, and also to have had some system of payment mocked up.
