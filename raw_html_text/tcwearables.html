Inspiration

Last week, watching A-company keynote, in the end, there is a band performing, are they wearing A-Watch? No! What's wrong?! A connected wearables system can connect people!

What it does

use case: concerts and sport games.

  1. "master-follower social interaction" mode: for engagement and interactions when singer or player shake weared wearable: send vibration or lights to attendee wearables
  2. "sentiment big data" mode: collect attendee wearable sensor data time series (accelerometer, etc) during the concert or game, for event production adjustment, or advertisement deployment.

How I built it

(1). server build with nodejs, express, socket.io (2). intel edison + seeedstudio xadow wearable kit + 3D Printed watch case you3Dyy67it running nodejs code talk to server, sending sensor data, or receiving master data to perform buzz, light or other actuators. (3). pebble code to connect to server doing the same as (2) (4). Myo connect to server doing the same as (2)

Challenges I ran into

(1) time constraint , did not finish intel edison + Myo Armband through BLE communications (2) time constraint, did not finish Android wear running crosswalk enabled web app
