Inspiration

We wanted to make music listening a more interactive and inclusive social experience.

What it does

Allows users to search and listen to songs on one platform and have a public voting system to allow friends of the listener to participate in the decision making on the next song in the queue.

How I built it

We built it with blood, sweat, lack of sleep, and tears.

Challenges I ran into

Communication between client and server (multiple handshakes - resolved), getting the music API to adapt to our page.

Accomplishments that I'm proud of

We have a usable product, we just have to fine tune things.

What I learned

I learned different APIs, front-end, server and client interaction.

What's next for Euphony

There were a lot of cool features that we wanted to put in, but did not have the time to do so. For example: a party queue, a local listening room, and user driven recommendations.
