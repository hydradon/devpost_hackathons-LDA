Inspiration

When thinking about the sponsor prize, we thought it would be cool to try and make something that could be used in retail. We thought making a system which could announce deals and offers which only people with their phones on could get as their phone would decode the deal so it was usable.

After some adjustments, we decided to make it for messaging.

What it does

It allows the user to send and decode ultrasonic messages.

How we built it

We used React and Semantic UI to create the interface. Then we used the web API audioContext to encode and decode the sounds.

Challenges we ran into

Getting the decoder to understand the sounds correctly.

Accomplishments that we're proud of

We managed to get it to send a message from one computer and receive it on another!

What we learned

Some of us had very little experience in JavaScript, web API audioContext, and React, so this was a learning curve for all of us!

What's next for Talking 2.0

Allow users to adjust to their own frequency range.
