Inspiration

Medical bills are not transparent and we wanted to make them more transparent

What it does

It uses CMS data to provide average procedural costs for patients while also creating a community where patients can discuss these procedures, and what to expect before and after

How we built it

We built the front ending using ReactJS and the back end data set was from Data.CMS.gov

Challenges we ran into

How we would make it profitable, and how to set us apart from other services that do similar things for patients

Accomplishments that we're proud of

We are proud to contribute towards advancing patient care with technology

What we learned

We learned a lot about planning business models, about CMS data, insurance and coverage, and how to build a website

What's next for ClarInsurance

Get more data from other private insurance providers to broaden our user base and help more people
