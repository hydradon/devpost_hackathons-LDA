• a wallet application that allows a recipient of a transaction to propagate a signed transaction from the sender

Inspiration

While returning from a client trip, our team joked around about the idea of the airline accepting crypto. We then realized that we do not have access to internet, which is require to propagate the transaction. Seeing people using their debit/credit cards we realized that all we need to do is mimic the UX of that payment. Basically allowing the recipient of the transaction to be the only one who needs internet connectivity.

What it does

weipay allows the sender of a transaction to sign a transaction offline, which they then allow the recipient of the transaction to propagate.

How we built it

This was built utilizing ethers.js and react-native.

Challenges we ran into

The main challenge was trying to build a decentralized mixer, or even integrating a reliable one for the time bing.

Accomplishments that we're proud of

We are very proud that we were able to bring the UX of debit and credit cards to the crypto world.

What we learned

We learned a lot about mixers and other forms of obfuscation.

What's next for weipay

Next we will be adding features that allow any ERC-20 to be viewed and transacted through the app. Another future feature is adding a mixer.
