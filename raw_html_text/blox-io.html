Inspiration

I/O Games such as Agar.io and Slither.io

What it does

Allows you to fight/defend against NPC monsters in real time with other users on separate computers

How we built it

Frontend: javascript Backend: socket.io and node.js

Challenges we ran into

Integrating visuals (front end) with code (back end), detecting collisions, github merging, deploying to server

Accomplishments that we're proud of

Collisions between players and NPCs, integrated visuals, actually having a presentable and usable product

What we learned

How to use Github, network sockets, server side vs client side coding differences.

What's next for Blox.io

To integrate a map/background image and consider collisions between players and NPCs. More people, more fun!
