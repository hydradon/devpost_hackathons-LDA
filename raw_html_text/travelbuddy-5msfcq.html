Inspiration

There are many times when you're not sure what you're eating when traveling! This is especially true at foreign countries when you don't speak the native tongue.

What it does

TravelBuddy fixes this problem by displaying a short snippet of whatever picture food / landmark you provide to it. It will also suggest nearby places to experience more of whatever is in your picture!

How I built it

We built this app using the Google App Engine, primarily Google Vision and Google Maps API. Additionally, we used the Wikipedia API to fetch short summaries.

Challenges I ran into

We had a difficult time figuring out what platform we wanted to develop on (mobile or web). We ultimately went with web after many hours of struggle with mobile.

Accomplishments that I'm proud of

We used machine learning!

What I learned

Projects should always be thoroughly thought out before executing. You save a lot more time.

What's next for TravelBuddy

We go mobile! Right now, users must upload a picture from a web browser to use our app. We hope to move to mobile so that users can use our app quicker while on the go.
